---
title: verbatra watch
description: Retraduce en cada cambio del origen hasta que lo interrumpas.
---
Vigila el archivo del locale de origen y vuelve a ejecutar una traducción en cada cambio asentado por el anti-rebote, hasta que lo interrumpas. `watch` ejecuta una traducción inicial al arrancar y luego mantiene al día tus locales de destino mientras guardas. Antes de ejecutarse, carga `.env.local` y luego `.env` desde el directorio de trabajo; una variable ya definida en el entorno real gana.

## Sinopsis

```bash
verbatra watch [flags]
```

## Flags

| Flag | Argumento | Por defecto | Efecto |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | directorio actual | resuelve la configuración y los archivos de locale desde este directorio |
| `--config` | `<path>` | busca uno | carga este archivo de configuración en lugar de buscar uno |
| `--debounce` | `<ms>` | `300` | espera estos milisegundos tras el último cambio antes de traducir; debe ser un entero positivo sin sufijo (un valor como `250ms`, `0` o `-1` es un error de uso, salida `2`) |
| `--concurrency` | `<n>` | `1` | traduce hasta `n` locales de destino a la vez por ejecución; debe ser un entero positivo |
| `--no-cache` | ninguno | desactivado (caché activada) | omite la [caché de memoria de traducción](/docs/the-cache) local (`verbatra.cache.json`) en cada ejecución |
| `--json` | ninguno | desactivado | imprime cada ejecución como un registro NDJSON (un objeto JSON por línea) en stdout |

Con `--json`, cada ejecución emite un registro [`WatchRunResult`](/docs/sdk): un éxito que lleva el resumen de la ejecución, o un fallo que lleva un código de error y un mensaje. La línea de arranque, el aviso de parada y el progreso de cada ejecución van a stderr, así que el flujo de stdout se mantiene como NDJSON limpio. Una ejecución fallida es solo un registro en el flujo: ni detiene el watcher ni cambia el código de salida.

`--concurrency` se aplica a la ejecución de cada ciclo. Como el presupuesto de tokens de cada ejecución es nuevo, un `--concurrency` mayor que `1` sigue fallando en cada ejecución con salida `2` cuando la configuración fija un presupuesto `maxTokens`. `--no-cache` omite la [caché de memoria de traducción](/docs/the-cache) en cada ciclo.

## Detener

La primera interrupción (Ctrl-C, o SIGTERM) deja terminar la ejecución en curso, detiene el watcher limpiamente y sale con `0`. Una segunda interrupción mientras esa parada sigue en curso fuerza la detención con salida `130`.

## Ejemplos

```bash
# watch and re-translate on each change
verbatra watch

# calmer in a busy editor: wait 1s after the last change, stream NDJSON
verbatra watch --debounce 1000 --json
```

## Códigos de salida

| Código | Significado |
| --- | --- |
| `0` | detenido limpiamente por una sola interrupción |
| `2` | no se pudo arrancar o detener: un error de configuración, un fallo de arranque o de parada, o un error de uso (incluido un `--debounce` inválido) |
| `130` | detenido a la fuerza por una segunda interrupción |
