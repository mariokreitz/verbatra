---
title: verbatra check
description: Melde, welche Keys pro Locale fehlen oder veraltet sind, ohne Dateien zu schreiben.
---
Melde pro Ziel-Locale, wie viele Keys fehlen, veraltet oder aktuell sind, und beende dich mit einem Code, auf den ein CI-Schritt verzweigen kann. `check` führt dasselbe Lesen und Diffen aus, das [`verbatra translate`](/docs/cli/translate) direkt vor dem Provider-Aufruf ausführen würde, und hört dann auf: kein Provider-Aufruf, keine geschriebenen Dateien, keine Aktualisierung der [Lock-Datei](/docs/the-lock-file) und kein API-Key nötig.

## Synopsis

```bash
verbatra check [flags]
```

## Flags

| Flag | Argument | Standard | Wirkung |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | aktuelles Verzeichnis | Konfiguration und Locale-Dateien von diesem Verzeichnis aus auflösen |
| `--config` | `<path>` | wird gesucht | diese Konfigurationsdatei laden, statt eine zu suchen |
| `--locales` | `<list>` | alle konfigurierten Ziele | kommagetrennte Teilmenge der Ziel-Locales, die geprüft wird |
| `--json` | keins | aus | die Prüf-Zusammenfassung als ein JSON-Objekt auf stdout ausgeben; Fehler gehen weiter auf stderr |

Ein Key fehlt, wenn die Quelle ihn hat und das Ziel nicht; er ist veraltet, wenn sich der Quellstring seit der letzten Übersetzung geändert hat. Ein Locale ist synchron, wenn es keines von beidem hat. Ein leerer oder unbekannter `--locales`-Wert wird mit Exit `2` abgelehnt, statt stillschweigend nichts zu prüfen; ein Tippfehler in einem Locale-Namen lässt ein CI-Gate also nie durch. Du willst die konkreten Key-Listen hinter den Zählern? Das ist [`verbatra diff`](/docs/cli/diff).

## Beispiele

```bash
# fail the build when translations drifted
verbatra check

# machine-readable status for two locales
verbatra check --locales de,fr --json
```

## Exit-Codes

| Code | Bedeutung |
| --- | --- |
| `0` | synchron: kein Locale hat einen fehlenden oder veralteten Key |
| `1` | nicht synchron: mindestens ein Locale hat einen fehlenden oder veralteten Key (der Bericht wird trotzdem ausgegeben) |
| `2` | konnte nicht laufen: ein Fehler des gesamten Laufs oder ein Verwendungsfehler (einschließlich eines leeren oder unbekannten `--locales`-Werts) |

Exit `1` heißt "nicht synchron", nicht Fehlschlag: der Lauf selbst war erfolgreich, das Ergebnis ist nur nicht komplett grün.
