# verbatra export

Schreibt die Strings, die übersetzt werden müssen, in eine gestylte Excel-Arbeitsmappe, die ein Mensch ausfüllen und dann für [`verbatra import`](/docs/cli/import) zurücksenden kann. Die Arbeitsmappe hat ein Blatt pro Ziel-Locale plus ein Anweisungsblatt; der Übersetzer bearbeitet eine einzige Spalte und alles andere ist gesperrt.

```bash
verbatra export
```

Standardmäßig schreibt dies `verbatra-translations.xlsx` in das Arbeitsverzeichnis und enthält nur die **fehlenden und geänderten** Strings - dieselben Schlüssel, die ein automatischer `translate`-Lauf an einen Provider senden würde. Siehe [Manuelle Übersetzung](/docs/manual-translation) für den vollständigen Hin- und Rückweg.

## Optionen

| Option | Beschreibung |
| --- | --- |
| `--cwd <path>` | löst Konfigurations- und Locale-Dateien aus diesem Verzeichnis auf |
| `--config <path>` | lädt diese Konfigurationsdatei, anstatt nach einer zu suchen |
| `--out <path>` | schreibt die Arbeitsmappe an diesen Pfad (Standard `verbatra-translations.xlsx`) |
| `--locales <list>` | kommagetrennte Teilmenge der Ziel-Locales (Standard alle konfigurierten) |
| `--include-unchanged` | exportiert auch bereits aktuelle Strings (standardmäßig aus) |
| `--json` | gibt das Export-Ergebnis als JSON aus |

## Ausgabe

Ohne `--json` gibt `export` den Ausgabepfad und eine Zeile pro Locale mit dessen Zeilenanzahl aus. Mit `--json` gibt es das Export-Ergebnis - den geschriebenen Pfad und die Zeilenanzahlen pro Locale - als ein JSON-Objekt auf stdout aus. Ein Lauf-weiter Fehler wird auf stderr geschrieben, sodass stdout sauber und unter `--json` pipebar bleibt.

## Beispiele

```bash
# write the workbook with missing and changed strings
verbatra export

# only the German and French sheets
verbatra export --locales de,fr

# include already up-to-date strings for full context
verbatra export --include-unchanged

# write to a specific path
verbatra export --out handoff/round-2.xlsx
```

## Exit-Codes

| Code | Bedeutung |
| --- | --- |
| `0` | die Arbeitsmappe wurde geschrieben |
| `2` | konnte nicht ausgeführt werden: ein Lauf-weiter Fehler (fehlende oder ungültige Konfiguration, nicht lesbare Quelle, unbekanntes Format oder eine beschädigte Lock-Datei) oder ein Nutzungsfehler |

Export ruft niemals einen Provider auf und benötigt keinen API-Schlüssel. Siehe [`verbatra import`](/docs/cli/import) für das Zurücklesen der ausgefüllten Arbeitsmappe.
