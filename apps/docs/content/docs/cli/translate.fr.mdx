---
title: verbatra translate
description: Traduit chaque locale cible une fois, puis termine.
---
Lance une passe de traduction sur chaque locale cible, puis termine. `translate` lit la locale source, la compare au [fichier de verrouillage](/docs/the-lock-file), n'envoie au fournisseur que les chaînes nouvelles ou modifiées, exécute les [vérifications d'intégrité](/docs/translation-safety), écrit les fichiers de locale et met à jour le verrou. Tout ce qui est encore à jour n'est jamais renvoyé, donc tu ne paies jamais deux fois pour une chaîne inchangée.

## Synopsis

```bash
verbatra translate [flags]
```

## Options

| Option | Argument | Défaut | Effet |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | répertoire courant | résout la config et les fichiers de locale depuis ce répertoire |
| `--config` | `<path>` | recherche automatique | charge ce fichier de config au lieu d'en chercher un |
| `--dry-run` | aucun | désactivé | prévisualise les changements sans appeler de fournisseur ni écrire de fichiers |
| `--prune` | aucun | l'option `prune` de la config, sinon désactivé | supprime les clés orphelines (présentes dans un fichier cible mais absentes de la source) du fichier écrit et du verrou |
| `--json` | aucun | désactivé | affiche le résumé d'exécution comme un seul objet JSON sur stdout (la [forme `RunSummary`](/docs/sdk)) ; les erreurs vont toujours sur stderr |

Par défaut, une exécution ne supprime rien : les clés orphelines sont signalées et laissées en place. `--prune` supprime exactement ces clés et rien d'autre, et remplace l'option `prune` de la config pour cette exécution. Avant de s'exécuter, `translate` charge `.env.local` puis `.env` depuis le répertoire de travail ; une variable déjà définie dans l'environnement réel l'emporte.

## Exemples

```bash
# translate once using the config it finds
verbatra translate

# preview the keys that would be pruned, spending nothing
verbatra translate --prune --dry-run
```

## Codes de sortie

| Code | Signification |
| --- | --- |
| `0` | chaque locale cible a réussi |
| `1` | l'exécution a terminé mais une ou plusieurs locales ont échoué |
| `2` | impossible de s'exécuter : une erreur globale (config, source, format, fournisseur ou verrou) ou une erreur d'utilisation |

Une locale qui échoue n'est pas fatale : l'exécution se termine, sort avec `1`, et le détail par locale se trouve dans le résumé.
