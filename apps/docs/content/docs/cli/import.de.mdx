---
title: verbatra import
description: Importiere eine ausgefüllte Arbeitsmappe zurück in die Locale-Dateien, mit denselben Sicherheitsprüfungen.
---
Lies eine von einer Übersetzerin ausgefüllte Arbeitsmappe zurück in deine Locale-Dateien. Jeder Wert läuft durch dieselben Prüfungen auf Platzhalter-Integrität und ICU-Integrität wie ein automatischer Lauf: Werte, die bestehen, werden geschrieben, und die [Lock-Datei](/docs/the-lock-file) wird aktualisiert; Werte, die durchfallen, werden zurückgehalten und gemeldet statt geschrieben. Eine kaputte Zelle landet also nie in einer Locale-Datei.

## Synopsis

```bash
verbatra import <workbook> [flags]
```

`<workbook>` ist der Pfad zur Arbeitsmappe, die [`verbatra export`](/docs/cli/export) erzeugt und die Übersetzerin ausgefüllt hat.

## Flags

| Flag | Argument | Standard | Wirkung |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | aktuelles Verzeichnis | Konfiguration und Locale-Dateien von diesem Verzeichnis aus auflösen |
| `--config` | `<path>` | wird gesucht | diese Konfigurationsdatei laden, statt eine zu suchen |
| `--dry-run` | keins | aus | validieren und melden, ohne Locale-Dateien zu schreiben oder das Lock zu aktualisieren |
| `--json` | keins | aus | die Lauf-Zusammenfassung als ein JSON-Objekt auf stdout ausgeben (dieselbe [`RunSummary`-Form](/docs/sdk) wie bei `translate`); Fehler gehen weiter auf stderr |

## Beispiele

```bash
# import the filled workbook
verbatra import verbatra-translations.xlsx

# validate and report, write nothing
verbatra import verbatra-translations.xlsx --dry-run
```

## Exit-Codes

| Code | Bedeutung |
| --- | --- |
| `0` | jedes Ziel-Locale war erfolgreich |
| `1` | der Lauf lief durch, aber ein oder mehrere Locales schlugen fehl |
| `2` | konnte nicht laufen: ein Fehler des gesamten Laufs (Konfiguration, Arbeitsmappe, Format oder Lock) oder ein Verwendungsfehler |

Der Vertrag entspricht [`verbatra translate`](/docs/cli/translate); eine manuelle Übergabe verzweigt in CI also genauso.
