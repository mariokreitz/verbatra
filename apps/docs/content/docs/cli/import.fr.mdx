# verbatra import

Relis un classeur rempli par un traducteur dans tes fichiers de locale. Import exécute les mêmes vérifications de placeholders et d'ICU que [`verbatra translate`](/docs/cli/translate) : chaque valeur est validée, celles qui passent sont écrites et le [fichier de verrouillage](/docs/the-lock-file) est mis à jour, et toute valeur qui échoue à une vérification est retenue et signalée plutôt qu'écrite.

```bash
verbatra import verbatra-translations.xlsx
```

L'argument est le chemin vers le classeur produit par [`verbatra export`](/docs/cli/export) et rempli par le traducteur. Seule la colonne `Translation` est lue ; les cellules vides sont ignorées, jamais écrites comme des chaînes vides. Voir [Traduction manuelle](/docs/manual-translation) pour l'aller-retour complet.

## Arguments

| Argument | Description |
| --- | --- |
| `<workbook>` | chemin vers le classeur rempli à importer |

## Options

| Option | Description |
| --- | --- |
| `--cwd <path>` | résoudre la configuration et les fichiers de locale depuis ce répertoire |
| `--config <path>` | charger ce fichier de configuration au lieu d'en rechercher un |
| `--dry-run` | valider et signaler sans écrire les fichiers de locale ni mettre à jour le verrou |
| `--json` | afficher le résumé de l'exécution au format JSON |

## Sortie

Import réutilise le résumé d'exécution de `translate` tel quel. Sans `--json`, il affiche un résumé lisible par un humain : une ligne par locale, puis un agrégat. Chaque ligne de locale indique combien de valeurs étaient `translated`, `unchanged`, `orphaned`, `invalid-ICU skipped` et `integrity-withheld`. Avec `--json`, il affiche la même [structure `RunSummary`](/docs/sdk) sous forme d'un seul objet JSON sur stdout.

Une valeur est **retenue** lorsqu'elle casse un placeholder, qu'elle est de l'ICU invalide, ou que sa source a changé depuis que le classeur a été exporté (détecté via la colonne cachée de hachage de source). Une valeur retenue laisse le fichier de locale intact afin que tu puisses la corriger et la réimporter. Voir [Sécurité de traduction](/docs/translation-safety) pour les vérifications.

## Exemples

```bash
# importer le classeur rempli
verbatra import verbatra-translations.xlsx

# valider et signaler, n'écrire rien
verbatra import verbatra-translations.xlsx --dry-run
```

## Codes de sortie

| Code | Signification |
| --- | --- |
| `0` | chaque locale cible a réussi |
| `1` | l'exécution s'est terminée mais une ou plusieurs locales ont échoué |
| `2` | impossible de s'exécuter : une erreur d'exécution globale (configuration manquante ou invalide, un classeur illisible ou invalide, format inconnu, ou un fichier de verrouillage corrompu), ou une erreur d'usage |

Le contrat des codes de sortie correspond à celui de [`verbatra translate`](/docs/cli/translate), de sorte qu'un transfert manuel s'embranche de la même manière en CI - voir [CI et automatisation](/docs/ci-and-exit-codes).
