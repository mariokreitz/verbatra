---
title: verbatra watch
description: Retraduit à chaque changement de la source jusqu'à interruption.
---
Surveille le fichier de la locale source et relance une traduction à chaque changement temporisé, jusqu'à ce que tu l'interrompes. `watch` lance une traduction initiale au démarrage, puis garde tes locales cibles à jour au fil de tes sauvegardes. Avant de s'exécuter, il charge `.env.local` puis `.env` depuis le répertoire de travail ; une variable déjà définie dans l'environnement réel l'emporte.

## Synopsis

```bash
verbatra watch [flags]
```

## Options

| Option | Argument | Défaut | Effet |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | répertoire courant | résout la config et les fichiers de locale depuis ce répertoire |
| `--config` | `<path>` | recherche automatique | charge ce fichier de config au lieu d'en chercher un |
| `--debounce` | `<ms>` | `300` | attend ce nombre de millisecondes après le dernier changement avant de traduire ; doit être un entier positif nu (une valeur comme `250ms`, `0` ou `-1` est une erreur d'utilisation, sortie `2`) |
| `--json` | aucun | désactivé | affiche chaque exécution comme un enregistrement NDJSON (un objet JSON par ligne) sur stdout |

Avec `--json`, chaque exécution émet un enregistrement [`WatchRunResult`](/docs/sdk) : un succès portant le résumé d'exécution, ou un échec portant un code d'erreur et un message. La ligne de démarrage et l'avis d'arrêt vont sur stderr, donc le flux stdout reste du NDJSON propre. Une exécution échouée n'est qu'un enregistrement sur le flux : elle n'arrête pas la surveillance et ne change pas le code de sortie.

## Arrêt

La première interruption (Ctrl-C ou SIGTERM) laisse l'exécution en cours se terminer, arrête proprement la surveillance et sort avec `0`. Une seconde interruption pendant que cet arrêt est encore en cours force l'arrêt avec le code `130`.

## Exemples

```bash
# watch and re-translate on each change
verbatra watch

# calmer in a busy editor: wait 1s after the last change, stream NDJSON
verbatra watch --debounce 1000 --json
```

## Codes de sortie

| Code | Signification |
| --- | --- |
| `0` | arrêté proprement par une seule interruption |
| `2` | impossible de démarrer ou de s'arrêter : une erreur de config, un échec au démarrage ou à l'arrêt, ou une erreur d'utilisation (y compris un `--debounce` invalide) |
| `130` | arrêté de force par une seconde interruption |
