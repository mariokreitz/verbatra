# verbatra watch

Surveillez le fichier source et réexécutez une traduction à chaque modification après temporisation, jusqu'à ce que vous l'interrompiez. Utilisez-le pendant que vous travaillez afin que les chaînes nouvelles et modifiées soient traduites au fur et à mesure que vous enregistrez.

```bash
verbatra watch
```

La surveillance continue de s'exécuter jusqu'à ce que vous l'arrêtiez. Une seule interruption (Ctrl-C) l'arrête proprement et quitte avec `0` ; une seconde interruption l'arrête de force et quitte avec `130`.

## Options

| Option | Description |
| --- | --- |
| `--cwd <path>` | résoudre la configuration et les fichiers de locale depuis ce répertoire |
| `--config <path>` | charger ce fichier de configuration au lieu d'en rechercher un |
| `--debounce <ms>` | attendre ce nombre de millisecondes après une modification avant de traduire (par défaut `300`) |
| `--json` | afficher chaque exécution sous forme d'un enregistrement NDJSON |

## Sortie

Sans `--json`, `watch` affiche un résumé lisible par un humain par exécution. Avec `--json`, il affiche un enregistrement NDJSON par exécution (un objet JSON par ligne) : soit un succès portant le résumé de l'exécution, soit un échec portant un code et un message d'erreur. Une exécution échouée est l'un de ces enregistrements sur le flux - elle n'arrête pas le surveillant et ne définit pas un code de sortie non nul. Voir la [structure `WatchRunResult` dans la référence du SDK](/docs/sdk).

## Exemples

```bash
# surveiller et retraduire à chaque modification
verbatra watch

# attendre 1 s après une modification avant de traduire
verbatra watch --debounce 1000

# diffuser un enregistrement NDJSON par exécution
verbatra watch --json
```

## Codes de sortie

| Code | Signification |
| --- | --- |
| `0` | arrêté proprement par une seule interruption |
| `2` | impossible de s'exécuter : la configuration n'a pas pu être chargée, ou une erreur d'usage |
| `130` | arrêté de force par une seconde interruption |

Contrairement à [`translate`](/docs/cli/translate), l'échec d'une exécution ne change pas le code de sortie - il est signalé comme un enregistrement de flux. Le code de sortie reflète uniquement la façon dont le surveillant s'est arrêté.
