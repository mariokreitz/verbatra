# verbatra import

Lee un libro rellenado por un traductor de vuelta a tus archivos de configuración regional. Import ejecuta las mismas comprobaciones de marcadores de posición e ICU que [`verbatra translate`](/docs/cli/translate): cada valor se valida, los que pasan se escriben y el [archivo de bloqueo](/docs/the-lock-file) se actualiza, y cualquier valor que falle una comprobación se retiene y se informa en lugar de escribirse.

```bash
verbatra import verbatra-translations.xlsx
```

El argumento es la ruta al libro producido por [`verbatra export`](/docs/cli/export) y rellenado por el traductor. Solo se lee la columna `Translation`; las celdas vacías se omiten, nunca se escriben como cadenas vacías. Consulta [Traducción manual](/docs/manual-translation) para ver el ciclo completo.

## Argumentos

| Argumento | Descripción |
| --- | --- |
| `<workbook>` | ruta al libro rellenado que se va a importar |

## Opciones

| Opción | Descripción |
| --- | --- |
| `--cwd <path>` | resuelve los archivos de configuración y de configuración regional desde este directorio |
| `--config <path>` | carga este archivo de configuración en lugar de buscar uno |
| `--dry-run` | valida e informa sin escribir archivos de configuración regional ni actualizar el bloqueo |
| `--json` | imprime el resumen de la ejecución como JSON |

## Salida

Import reutiliza el resumen de la ejecución de `translate` sin cambios. Sin `--json`, imprime un resumen legible para humanos: una línea por configuración regional, luego un agregado. Cada línea de configuración regional informa cuántos valores fueron `translated`, `unchanged`, `orphaned`, `invalid-ICU skipped` e `integrity-withheld`. Con `--json`, imprime la misma [forma de `RunSummary`](/docs/sdk) como un único objeto JSON en stdout.

Un valor se **retiene** cuando rompe un marcador de posición, es ICU no válido, o su origen cambió desde que se exportó el libro (detectado a través de la columna oculta de hash de origen). Un valor retenido deja el archivo de configuración regional intacto para que puedas corregirlo y volver a importar. Consulta [Seguridad de la traducción](/docs/translation-safety) para conocer las comprobaciones.

## Ejemplos

```bash
# importa el libro rellenado
verbatra import verbatra-translations.xlsx

# valida e informa, no escribe nada
verbatra import verbatra-translations.xlsx --dry-run
```

## Códigos de salida

| Código | Significado |
| --- | --- |
| `0` | cada configuración regional de destino tuvo éxito |
| `1` | la ejecución terminó pero una o más configuraciones regionales fallaron |
| `2` | no se pudo ejecutar: un error de ejecución completa (configuración ausente o no válida, un libro ilegible o no válido, formato desconocido, o un archivo de bloqueo corrupto), o un error de uso |

El contrato del código de salida coincide con el de [`verbatra translate`](/docs/cli/translate), por lo que una entrega manual bifurca de la misma manera en CI: consulta [CI y automatización](/docs/ci-and-exit-codes).
