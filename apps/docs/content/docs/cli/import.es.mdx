---
title: verbatra import
description: Importa un libro rellenado de vuelta a los archivos de locale, ejecutando las mismas comprobaciones de seguridad.
---
Lee un libro que un traductor rellenó de vuelta a tus archivos de locale. Cada valor pasa por las mismas comprobaciones de integridad de marcadores de posición e ICU que una ejecución automática: los valores que pasan se escriben y [el archivo de bloqueo](/docs/the-lock-file) se actualiza; los valores que fallan se retienen y se informan en lugar de escribirse, así que una celda mala nunca acaba en un archivo de locale.

## Sinopsis

```bash
verbatra import <workbook> [flags]
```

`<workbook>` es la ruta al libro producido por [`verbatra export`](/docs/cli/export) y rellenado por el traductor.

## Flags

| Flag | Argumento | Por defecto | Efecto |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | directorio actual | resuelve la configuración y los archivos de locale desde este directorio |
| `--config` | `<path>` | busca uno | carga este archivo de configuración en lugar de buscar uno |
| `--dry-run` | ninguno | desactivado | valida e informa sin escribir archivos de locale ni actualizar el bloqueo |
| `--json` | ninguno | desactivado | imprime el resumen de la ejecución como un objeto JSON en stdout (la misma [forma `RunSummary`](/docs/sdk) que `translate`); los errores siguen yendo a stderr |

## Ejemplos

```bash
# import the filled workbook
verbatra import verbatra-translations.xlsx

# validate and report, write nothing
verbatra import verbatra-translations.xlsx --dry-run
```

## Códigos de salida

| Código | Significado |
| --- | --- |
| `0` | todos los locales de destino tuvieron éxito |
| `1` | la ejecución terminó pero uno o más locales fallaron |
| `2` | no se pudo ejecutar: un error de toda la ejecución (configuración, libro, formato o bloqueo) o un error de uso |

El contrato coincide con el de [`verbatra translate`](/docs/cli/translate), así que una entrega manual ramifica igual en CI.
