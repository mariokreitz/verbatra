---
title: verbatra diff
description: Affiche les clés qui seraient ajoutées, retraduites ou orphelines par locale sans écrire de fichiers.
---
Liste, par locale cible, les clés exactement en attente : celles qu'une exécution ajouterait (manquantes dans la cible), celles qu'elle retraduirait (source modifiée depuis la dernière traduction) et celles désormais orphelines (dans la cible mais disparues de la source). `diff` exécute la même lecture et la même comparaison que [`verbatra translate`](/docs/cli/translate) effectue juste avant d'appeler un fournisseur, puis s'arrête : aucun appel de fournisseur, aucun fichier écrit, aucune mise à jour du [fichier de verrouillage](/docs/the-lock-file) et aucune clé d'API requise.

## Synopsis

```bash
verbatra diff [flags]
```

## Options

| Option | Argument | Défaut | Effet |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | répertoire courant | résout la config et les fichiers de locale depuis ce répertoire |
| `--config` | `<path>` | recherche automatique | charge ce fichier de config au lieu d'en chercher un |
| `--locales` | `<list>` | toutes les cibles configurées | sous-ensemble de locales cibles à comparer, séparées par des virgules |
| `--json` | aucun | désactivé | affiche le résumé de comparaison comme un seul objet JSON sur stdout ; les erreurs vont toujours sur stderr |

Une locale a des changements en attente quand elle a au moins une clé à ajouter ou à retraduire. Les clés orphelines sont signalées mais ne comptent jamais comme en attente, parce qu'une exécution par défaut ne les supprime pas (seul [`verbatra translate --prune`](/docs/cli/translate) le fait). Une valeur `--locales` vide ou inconnue est rejetée avec la sortie `2` plutôt que de ne rien comparer en silence. Là où [`verbatra check`](/docs/cli/check) répond "combien a dérivé" avec des comptes, `diff` répond "quelles clés" avec les listes.

## Exemples

```bash
# list the pending keys per locale (exit 1 if any are pending)
verbatra diff

# machine-readable key lists for tooling or a PR comment
verbatra diff --json
```

## Codes de sortie

| Code | Signification |
| --- | --- |
| `0` | aucun changement en attente : aucune locale n'a de clé à ajouter ou à retraduire |
| `1` | changements en attente : au moins une locale a une clé manquante ou modifiée (les clés orphelines seules ne produisent jamais `1`) |
| `2` | impossible de s'exécuter : une erreur globale, ou une erreur d'utilisation (y compris une valeur `--locales` vide ou inconnue) |
