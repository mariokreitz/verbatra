Verbatra Studio is a local web dashboard over your verbatra project: a live view of translation status, pending changes, configuration, lock-file drift, and history, without ever calling a provider or writing a file. Start it with:

```bash
verbatra studio
```

## Local-only and read-only

Studio binds to `127.0.0.1` only: it is never reachable from another machine, and there is no option to change that. Every request is checked against a strict Host and Origin allowlist, and every page and API call requires the per-session token printed in the startup URL, compared in constant time so a wrong guess cannot be timed. Studio sends no CORS headers, so a browser tab on another origin cannot read it even if it tried. There is no state-changing endpoint anywhere in Studio: it never calls `translate`, never runs `watch`, and never imports or exports a workbook. Everything you see comes from the same read-only computations as [`verbatra check`](/docs/cli/check) and [`verbatra diff`](/docs/cli/diff).

## Command reference

| Option | Description |
| --- | --- |
| `--cwd <path>` | resolve config and locale files from this directory |
| `--config <path>` | load this config file instead of searching for one |
| `--port <n>` | override the default Studio port (must be an integer from 1 to 65535) |

`verbatra studio` loads `.env.local` and `.env` exactly like every other command, then resolves your config once, before it ever touches the network. A missing or invalid config exits `2` with the same rendered error every other command uses; Studio itself is never started.

## The printed URL

Once Studio is listening, it prints one line to stdout:

```
Verbatra Studio running at http://127.0.0.1:5849/?token=<token>
```

Open that exact URL. The token in the query string bootstraps a session cookie for the browser tab; without it (or with the wrong one) every request is rejected. Nothing is ever printed before the server has successfully bound its port, so a failed startup never leaks a URL or a token.

## Restart on config change

The config is resolved once, at startup, and every view for the life of that process is built from that same resolved config. If you change `verbatra.config.ts` (a new target locale, a different provider, a different file pattern), stop Studio and start it again to pick up the change. Project files themselves (your source and target locale files, and the lock file) are read fresh on every request, so editing them and reloading the page always shows current data; Studio does not yet push an automatic refresh when a file changes on disk.

## The views

- **Status**: per-locale missing, stale, and up-to-date counts, the same computation as `verbatra check`.
- **Diff**: the exact keys that would be added, re-translated, or are orphaned per locale, the same computation as `verbatra diff`. Long key lists are capped in the view (with a "showing N of total" note) so a very large catalog stays responsive; filtering still searches the full list.
- **Config and glossary**: a redacted projection of your resolved config (source and target locales, format, file pattern, provider id, and the safe scalar options) plus your glossary terms, if any. Free-form values that could carry a pasted secret are never included.
- **Lock drift**: whether a lock file exists, its version, and its per-locale drift against the current source and target files, without writing to it.
- **History**: the commit history for your watched locale files, read via `git log` with a bounded commit count and no `--follow`. A project with no git repository, or with git unavailable, shows history as unavailable rather than an error. A shallow clone's truncated history is also not an error: it simply shows what the clone has.

## Installation

`@verbatra/studio` is Studio's own package. If it is not installed, `verbatra studio` exits `2` with:

```
Verbatra Studio requires @verbatra/studio. Install it with: pnpm add -D @verbatra/studio
```

## Exit codes

| Code | Meaning |
| --- | --- |
| `0` | Studio was stopped cleanly by a single interrupt (Ctrl-C) |
| `2` | could not run: a whole-run error (missing or invalid config, `@verbatra/studio` not installed, or the port is already in use), or a usage error (an out-of-range or non-integer `--port`) |
| `130` | force-stopped by a second interrupt |
