Starte Verbatra Studio, ein lokales Web-Dashboard über deinem verbatra-Projekt: Übersetzungsstatus, die Review-Warteschlange, Lauf-Aktivität und deine aufgelösten Einstellungen, alles live über den Dateien auf der Platte. Das Bearbeiten deiner eigenen Locale-Dateien funktioniert von Haus aus; nur kostenpflichtige Provider-Aufrufe sind Opt-in, hinter `--allow-spend`. Vor dem Start lädt `studio` `.env.local` und dann `.env` aus dem Arbeitsverzeichnis und löst deine Konfiguration einmal auf; ein Konfigurationsfehler endet mit `2`, bevor irgendein Server startet.

## Synopsis

```bash
verbatra studio [flags]
```

## Flags

| Flag | Argument | Standard | Wirkung |
| --- | --- | --- | --- |
| `--cwd` | `<path>` | aktuelles Verzeichnis | Konfiguration und Locale-Dateien von diesem Verzeichnis aus auflösen |
| `--config` | `<path>` | wird gesucht | diese Konfigurationsdatei laden, statt eine zu suchen |
| `--port` | `<n>` | `5849` | auf diesem Port lauschen; muss eine Ganzzahl von 1 bis 65535 sein |
| `--allow-spend` | keins | aus | Studio erlauben, einen Übersetzungs-Provider aufzurufen (einen Key neu übersetzen, ausstehende Änderungen übersetzen) |

Wenn das Flag `--allow-spend` fehlt, liest Studio stattdessen die Umgebungsvariable `VERBATRA_STUDIO_ALLOW_SPEND`: `1`, `true`, `yes` oder `on` (Groß- und Kleinschreibung egal) zählt als an. Das CLI-Flag gewinnt immer über die Umgebungsvariable. Ohne beides kontaktiert Studio nie einen Provider; lokales Bearbeiten braucht kein Flag.

## Die ausgegebene URL

Studio bindet sich nur an `127.0.0.1`. Sobald es lauscht, gibt es eine Zeile auf stdout aus:

```
Verbatra Studio running at http://127.0.0.1:5849/?token=<token>
```

Öffne genau diese URL: das Token im Query-String startet deine Browser-Sitzung, und jede Anfrage ohne Token wird abgelehnt. Vor dem erfolgreichen Binden des Ports wird nichts ausgegeben; ein fehlgeschlagener Start leakt also nie eine URL oder ein Token.

## Installation

`@verbatra/studio` ist ein eigenes Paket, erreicht über einen dynamischen Import, damit der Rest der CLI auch ohne es funktioniert. Ist es nicht installiert, endet `verbatra studio` mit `2` und:

```
Verbatra Studio requires @verbatra/studio. Install it with: pnpm add -D @verbatra/studio
```

## Beispiele

```bash
# start Verbatra Studio on the default port
verbatra studio

# enable the provider-calling actions, on a specific port
verbatra studio --allow-spend --port 6000
```

## Exit-Codes

| Code | Bedeutung |
| --- | --- |
| `0` | sauber gestoppt durch einen einzelnen Interrupt (Ctrl-C oder SIGTERM) |
| `1` | Stoppen fehlgeschlagen: das Schließen des Servers warf beim Herunterfahren einen Fehler |
| `2` | konnte nicht starten: ein Konfigurationsfehler, `@verbatra/studio` nicht installiert, der Port belegt oder ein Verwendungsfehler (ein `--port` außerhalb des Bereichs oder keine Ganzzahl) |
| `130` | zwangsbeendet durch einen zweiten Interrupt, während das Herunterfahren noch lief |
