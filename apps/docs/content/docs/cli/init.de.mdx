# verbatra init

Erstellt eine `verbatra.config.ts` und eine `.env.example` und fügt `.env` und `.env.local` zu deiner `.gitignore` hinzu, damit ein echter Schlüssel niemals committet wird. Mit einem angeschlossenen Terminal und ohne `--yes` fragt `init` nach den Werten, für die es keine Standardwerte gibt; andernfalls läuft es nicht-interaktiv und verwendet die Standardwerte.

```bash
verbatra init --provider anthropic
```

## Optionen

| Option | Beschreibung |
| --- | --- |
| `--cwd <path>` | schreibt die Konfigurations- und Umgebungsdateien in dieses Verzeichnis |
| `--provider <id>` | Übersetzungs-Provider: `anthropic`, `openai`, `gemini` oder `deepl` (erforderlich, sofern nicht abgefragt) |
| `--source <locale>` | Locale, in dem deine Quell-Strings geschrieben sind (Standard `en`) |
| `--targets <locales>` | kommagetrennte Locales, in die übersetzt werden soll (Standard `de`) |
| `--path <pattern>` | Locale-Dateimuster, das das `{locale}`-Token enthält (Standard `locales/{locale}.json`) |
| `--yes` | überspringt Aufforderungen und akzeptiert die Standardwerte |
| `--force` | überschreibt eine bestehende Konfiguration oder `.env.example` |

Der Provider ist der eine Wert, für den `init` keinen Standardwert hat. Ohne ein Terminal zum Abfragen (in CI oder mit `--yes`) übergib `--provider`. Die anderen Werte fallen auf ihre Standardwerte zurück, wenn sie nicht angegeben werden.

## Bestehende Dateien

Standardmäßig überschreibt `init` niemals: Wenn eine `verbatra.config.ts` oder `.env.example` bereits existiert, bleibt sie unberührt und der Befehl ist trotzdem erfolgreich. Übergib `--force`, um eine bestehende Datei zu überschreiben.

## Beispiele

```bash
# create config + .env example, prompting for the rest
verbatra init --provider anthropic

# non-interactive, accept all defaults
verbatra init --provider deepl --yes
```

## Exit-Codes

| Code | Bedeutung |
| --- | --- |
| `0` | die Dateien wurden geschrieben oder sicher übersprungen, weil sie bereits existieren (ohne `--force`) |
| `2` | ein Nutzungsfehler: ohne ein Terminal zum Abfragen wurde kein `--provider` angegeben, oder die zusammengesetzte Konfiguration ist bei der Validierung fehlgeschlagen |

Setze nach `init` den API-Schlüssel des Providers und führe [`verbatra translate`](/docs/cli/translate) aus. Siehe [Erste Schritte](/docs) für die vollständige Anleitung.
