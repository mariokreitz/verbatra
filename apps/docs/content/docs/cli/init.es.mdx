---
title: verbatra init
description: Genera una configuración de verbatra y un ejemplo de .env para tu proyecto.
---
Genera un `verbatra.config.ts` y un `.env.example`, y añade `.env` y `.env.local` a tu `.gitignore` para que una clave real nunca aterrice en un commit. Con un terminal conectado y sin `--yes`, `init` solicita los valores que no puede establecer por defecto; de lo contrario, se ejecuta de forma no interactiva y toma los valores por defecto.

¿Empiezas un proyecto pequeño? Gemini tiene un nivel gratuito, así que es un buen valor por defecto con el que generar:

```bash
verbatra init --provider gemini
```

## Opciones

| Opción | Descripción |
| --- | --- |
| `--cwd <path>` | escribe los archivos de configuración y de entorno en este directorio |
| `--provider <id>` | proveedor de traducción: `anthropic`, `openai`, `gemini` o `deepl` (obligatorio salvo que se solicite) |
| `--source <locale>` | locale en el que están escritas tus cadenas de origen (por defecto `en`) |
| `--targets <locales>` | locales a los que traducir, separados por comas (por defecto `de`) |
| `--path <pattern>` | patrón de archivo de locale que contiene el token `{locale}` (por defecto `locales/{locale}.json`) |
| `--yes` | omite las solicitudes y acepta los valores por defecto |
| `--force` | sobrescribe una configuración o un `.env.example` existente |

El proveedor es el único valor que `init` no puede adivinar. Sin un terminal para solicitar (en CI, o con `--yes`), pasa `--provider`. Todo lo demás recurre a su valor por defecto cuando lo omites.

## Archivos existentes

Por defecto, `init` nunca sobrescribe. Si ya hay un `verbatra.config.ts` o un `.env.example`, se deja intacto y el comando aun así tiene éxito. Pasa `--force` cuando realmente quieras sobrescribir.

## Ejemplos

```bash
# crea config + .env de ejemplo, solicitando el resto
verbatra init --provider gemini

# no interactivo, acepta todos los valores por defecto
verbatra init --provider deepl --yes

# establece los locales de origen y destino de antemano
verbatra init --provider gemini --source en --targets de,es,fr

# apunta a una disposición de archivos de locale no predeterminada
verbatra init --provider gemini --path src/locales/{locale}.json

# genera en otro directorio
verbatra init --provider gemini --cwd packages/app

# sobrescribe una configuración y un .env de ejemplo existentes
verbatra init --provider gemini --force
```

## Códigos de salida

| Código | Significado |
| --- | --- |
| `0` | los archivos se escribieron, o se omitieron de forma segura porque ya existen (sin `--force`) |
| `2` | un error de uso: no se dio ningún `--provider` sin un terminal para solicitar, o la configuración ensamblada no superó la validación |

Una vez que `init` termina, establece la clave de API del proveedor y ejecuta [`verbatra translate`](/docs/cli/translate). Consulta [la guía rápida](/docs) para ver el recorrido completo.
