# API programmatique

La [référence du SDK](/docs/sdk) documente chaque fonction et chaque type. Cette page en est le compagnon orienté tâches : quelques modèles de bout en bout pour utiliser verbatra dans du code. Tourne-toi vers le SDK lorsque tu veux de la traduction à l'intérieur d'un script, d'une étape de build ou d'une tâche CI, avec un contrôle total sur ce qui se passe autour.

## Traduire dans un script

Charge la configuration, exécute une traduction et signale l'essentiel. Les problèmes sur toute l'exécution lèvent une `SdkError` ; les résultats par locale sont des données sur le résumé.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig();
const summary = await translate({ config });

console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Conditionner une étape de build ou de CI au résultat

Le résumé t'indique ce qui a changé et ce qui a échoué sans analyser de sortie. Fais échouer l'étape lorsqu'une locale a échoué, et inspecte le détail par locale pour journaliser pourquoi.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const summary = await translate({ config: await loadConfig() });

for (const locale of summary.locales) {
  if (locale.status === "failed") {
    console.error(`${locale.locale}: ${locale.error?.code} ${locale.error?.message}`);
  }
}

if (summary.failed.length > 0) {
  process.exitCode = 1;
}
```

## Vérifier sans écrire

`check` lit et compare sans appeler de fournisseur ni écrire de fichiers, et signale la dérive par locale répartie en décomptes de clés manquantes, obsolètes et à jour. Son drapeau `inSync` est vrai exactement lorsque rien n'est manquant ni obsolète, c'est donc le garde-fou naturel pour la CI.

```ts
import { check, loadConfig } from "@verbatra/sdk";

const summary = await check({ config: await loadConfig() });

if (!summary.inSync) {
  const drifted = summary.locales.filter((locale) => !locale.inSync).map((locale) => locale.locale);
  console.error(`Out of sync: ${drifted.join(", ")}. Run verbatra translate.`);
  process.exitCode = 1;
}
```

`translate({ dryRun: true })` est une alternative lorsque tu veux le résumé d'exécution complet (les clés qui seraient traduites) plutôt que seulement les décomptes de dérive.

`diff` est la commande sœur en lecture seule de `check` : il exécute le même calcul mais renvoie les listes de clés réelles (`missing`, `changed`, `orphaned`) par locale au lieu de décomptes, avec le même garde-fou `hasPendingChanges`. Tourne-toi vers lui lorsque tu as besoin de savoir quelles clés, et pas seulement combien.

```ts
import { diff, loadConfig } from "@verbatra/sdk";

const summary = await diff({ config: await loadConfig() });

if (summary.hasPendingChanges) {
  for (const locale of summary.locales) {
    if (locale.hasPendingChanges) {
      console.log(`${locale.locale}: add ${locale.missing.join(", ")}; re-translate ${locale.changed.join(", ")}`);
    }
  }
  process.exitCode = 1;
}
```

## Confier des chaînes à un traducteur humain

Exporte les chaînes qui ont besoin d'être traduites dans un classeur Excel, puis réimporte le fichier rempli. `importWorkbook` renvoie le même `RunSummary` que `translate` et exécute les mêmes vérifications de placeholders et d'ICU, de sorte qu'un transfert manuel s'intègre aux mêmes rapports et garde-fous qu'une exécution automatisée. Voir [Traduction manuelle](/docs/manual-translation).

```ts
import { exportWorkbook, importWorkbook, loadConfig } from "@verbatra/sdk";

const config = await loadConfig();

// Write a workbook of the strings that need translating.
const exported = await exportWorkbook({ config });
console.log(`Wrote ${exported.path}`);

// ...later, after a translator fills and returns the file, import it back.
const summary = await importWorkbook({ config, workbook: exported.path });
console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Retraduire au fur et à mesure que la source change

`watch` appelle ton `onRun` une fois par changement débondi et renvoie un contrôleur que tu arrêtes proprement.

```ts
import { loadConfig, watch } from "@verbatra/sdk";

const controller = await watch({
  config: await loadConfig(),
  onRun: (result) => {
    if (result.status === "failed") {
      console.error(`run failed: ${result.error.code}`);
    }
  },
});

process.on("SIGINT", () => {
  void controller.stop();
});
```

## Une configuration sans fichier

`loadConfig` accepte un `configOverride` en mémoire, de sorte que tu peux piloter verbatra entièrement depuis le code sans aucun `verbatra.config.ts` sur le disque.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig({
  configOverride: {
    sourceLocale: "en",
    targetLocales: ["de", "fr"],
    format: "i18next-json",
    files: { pattern: "locales/{locale}.json" },
    provider: { id: "anthropic", options: { model: "claude-sonnet-4-6", maxTokens: 4096 } },
  },
});

await translate({ config });
```

Pour la liste complète des fonctions, les formes `RunSummary` et `LocaleSummary`, et chaque code `SdkError`, voir la [référence du SDK](/docs/sdk). Pour le contrat des codes de sortie lorsque tu exécutes la CLI en CI à la place, voir [CI et codes de sortie](/docs/ci-and-exit-codes).
