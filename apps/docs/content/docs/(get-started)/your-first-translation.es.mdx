title: Tu primera traducción

---
La CLI está instalada, la configuración está generada y la clave de tu proveedor está en el entorno (si no, empieza por la [guía rápida](/docs)). Es hora de traducir.

## Ejecútalo

Un solo comando traduce cada locale de destino y sale:

```bash
verbatra translate
```

Esto es lo que ocurre internamente: verbatra lee tu locale de origen, lo compara con lo que tradujo la última vez y envía a tu proveedor solo las cadenas nuevas y modificadas. Todo lo que sigue vigente se deja intacto. Cada archivo de destino se escribe en su sitio, así que `locales/de.json` y sus compañeros aparecen (o se actualizan) justo al lado de tu archivo de origen.

## Lee el resumen de la ejecución

Cada ejecución termina con un resumen. Obtienes el titular (qué locales tuvieron éxito, cuáles fallaron) y un desglose por locale: claves traducidas, claves sin cambios, claves huérfanas (presentes en un destino pero desaparecidas del origen) y cualquier aviso del proveedor. Que un locale falle no detiene el resto; aparece en la línea de ese locale con un código y un mensaje.

¿Lo quieres en formato legible por máquina? Añade `--json` y verbatra imprime todo el resumen como un único objeto JSON:

```bash
verbatra translate --json
```

Consulta [CI y códigos de salida](/docs/ci-and-exit-codes) para conocer la forma del JSON y el contrato de códigos de salida.

## Previsualiza antes de gastar un token

Para ver exactamente qué haría una ejecución sin llamar a un proveedor ni escribir un solo archivo, haz una ejecución en seco:

```bash
verbatra translate --dry-run
```

Obtienes el mismo resumen, con la lista de las claves que se traducirían, pero no se envía nada y no se escribe nada. Es la forma segura de comprobar una ejecución antes de que te cueste nada, algo especialmente práctico con una clave de nivel gratuito.

## Las reejecuciones solo tocan lo que cambió

Esta es la parte que hace que verbatra sea barato de ejecutar a menudo. Ejecuta `verbatra translate` de nuevo sin editar tu origen y nada llega al proveedor: todas las claves ya están vigentes. Cambia una cadena de origen y solo esa clave se vuelve a traducir en la siguiente ejecución.

El truco es un archivo de bloqueo versionado (`verbatra.lock.json`) que registra lo que se tradujo. Cada ejecución lo compara, así que el proveedor se llama únicamente para lo que realmente cambió. Consulta [Cómo funciona](/docs/how-it-works) para ver el flujo completo.

<Callout title="Versiona el archivo de bloqueo">
verbatra escribe `verbatra.lock.json` en la primera ejecución. Versiónalo junto a tus archivos de locale para que cada máquina, y tu CI, comparen con la misma línea base. Consulta [El archivo de bloqueo](/docs/the-lock-file).
</Callout>

## Traduce mientras trabajas

¿No quieres reejecutar a mano? Observa el origen y deja que verbatra vuelva a traducir con cada cambio:

```bash
verbatra watch
```

Sigue ejecutándose hasta que lo detengas con Ctrl-C.
