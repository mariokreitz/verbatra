# Testing and quality

verbatra is built to be depended on. Every change runs through a layered test suite, and the results are public so you can check them before you upgrade.

## What the badges mean

- **CI**: lint, type checks, unit tests, and the production build run on every pull request and on `main`, across two Node versions. CI enforces 90 percent coverage on lines, functions, statements, and branches.
- **E2E**: the published `@verbatra/cli` and `@verbatra/sdk` tarballs are installed into a throwaway project and driven through the real binary, so packaging and bundling regressions are caught before release.
- **Coverage**: the latest coverage reported to Codecov from the CI run on `main`.

## How it is tested

- **Unit tests** (Vitest) cover the domain logic, format adapters, providers, and CLI in every package.
- **End-to-end tests** install the real tarballs and exercise the full command surface: `init`, `check`, `diff`, `export`, `import`, `translate`, and `watch`. A no-key tier runs on every pull request; a live tier translates against a real provider nightly and after each merge.
