## Was die Badges bedeuten

- **CI**: Lint, Typprüfungen, Unit-Tests und der Produktions-Build laufen bei jedem Pull Request und auf `main`, über zwei Node-Versionen. Der `e2e`-Job in diesem Workflow installiert die veröffentlichten Tarballs von `@verbatra/cli` und `@verbatra/sdk` in ein Wegwerf-Projekt und führt sie über die echte Binary aus, sodass Packaging- und Bundling-Regressionen vor dem Release gefunden werden. CI erzwingt 90 Prozent Coverage für Zeilen, Funktionen, Anweisungen und Branches.
- **Coverage**: Die aktuelle Coverage, die aus dem CI-Lauf auf `main` an Codecov gemeldet wird.

## Wie getestet wird

- **Unit-Tests** (Vitest) decken die Domänenlogik, die Format-Adapter, die Provider und die CLI in jedem Paket ab.
- **End-to-End-Tests** installieren die echten Tarballs und üben die gesamte Befehlsfläche aus: `init`, `check`, `diff`, `export`, `import`, `translate` und `watch`. Eine Stufe ohne Schlüssel läuft bei jedem Pull Request; eine Live-Stufe übersetzt nächtlich und nach jedem Merge gegen einen echten Provider.
