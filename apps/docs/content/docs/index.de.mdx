## Schnellstart

Vier Schritte bringen dich vom leeren Projekt zu übersetzten Locale-Dateien. Die CLI ist der schnelle Weg; alles, was sie tut, ist auch über [das SDK](/docs/sdk) verfügbar.

### 1. Installieren

verbatra ist eine Dev-Abhängigkeit (hol es dir über die Installationskarte oben oder mit `pnpm add -D @verbatra/cli`). Es benötigt Node.js `>=22.14.0`, und du rufst das Binary über deinen Paketmanager auf: `pnpm verbatra ...`, `npx verbatra ...` oder `yarn verbatra ...`.

### 2. Eine Konfiguration erzeugen

`init` schreibt eine `verbatra.config.ts`, eine `.env.example` und ignoriert deine `.env` per gitignore, damit ein echter Schlüssel niemals in einem Commit landet.

```bash
verbatra init --provider gemini
```

Mit einem angeschlossenen Terminal fragt es nach allem, was es nicht per Default setzen kann; füge `--yes` hinzu, um die Defaults zu übernehmen (Quelle `en`, Ziel `de`, Dateien `locales/{locale}.json`) und die Abfragen zu überspringen.

### 3. Deinen API-Schlüssel setzen

Schlüssel kommen aus der Umgebung, niemals aus der Konfiguration. Jeder Provider liest genau eine Variable. Für Gemini ist das `GEMINI_API_KEY`.

```bash
export GEMINI_API_KEY=your-key-here
```

verbatra lädt vor dem Lauf zuerst `.env.local`, dann `.env` aus dem Arbeitsverzeichnis, und eine Variable, die bereits in deiner Shell steht, gewinnt immer. Siehe [Provider](/docs/providers) für die Variable jedes Providers.

<Callout title="Kleines Projekt oder Nebenprojekt? Nutze Gemini kostenlos.">
Gemini hat eine wirklich kostenlose API-Stufe, was es zum einfachsten Weg macht, ganz ohne Kosten zu starten. Hol dir einen Schlüssel bei [Google AI Studio](https://aistudio.google.com/apikey), setze `GEMINI_API_KEY` und führe `verbatra init --provider gemini` aus. Die kostenlose Stufe hat Anfrage-Limits pro Minute und pro Tag, drossle also eine große erstmalige Übersetzung (eine Locale nach der anderen reicht völlig). Siehe [Provider](/docs/providers) für die Abwägungen gegenüber Anthropic, OpenAI und DeepL.
</Callout>

### 4. Übersetzen

```bash
verbatra translate
```

verbatra liest die Quell-Locale, vergleicht sie mit dem, was es beim letzten Mal übersetzt hat, und schickt nur die neuen und geänderten Strings an Gemini. Jede Zieldatei wird an Ort und Stelle geschrieben, neben deiner Quelle. Führe es erneut aus, ohne etwas zu ändern, und nichts wird gesendet: Jeder Key ist bereits aktuell.
