# Ajouter une langue

L'ajout d'une langue est une modification de configuration d'une seule ligne suivie d'une exécution de traduction. verbatra considère la nouvelle locale comme manquant de tout, donc la première exécution la remplit complètement ; les exécutions ultérieures restent incrémentielles.

## Ajouter la locale à ta configuration

Ajoute la nouvelle locale à `targetLocales` dans `verbatra.config.ts`. Elle ne doit pas être ta `sourceLocale` :

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"], // "fr" added
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "anthropic",
    options: {
      model: "<your-model>",
      maxTokens: 4096,
    },
  },
});
```

## Traduire

Exécute une traduction :

```bash
verbatra translate
```

La nouvelle locale n'a pas encore de fichier cible, donc verbatra la considère comme vide et y traduit chaque clé source, en écrivant le fichier à partir de `files.pattern` (par exemple `locales/fr.json`). Tes locales existantes restent inchangées, sauf si leurs chaînes source ont été modifiées, car le fichier de verrouillage les enregistre toujours comme actuelles.

Prévisualise d'abord avec `verbatra translate --dry-run` si tu souhaites voir les clés de la nouvelle locale avant tout appel de fournisseur.

## Langues avec plus de catégories de pluriel

Certaines langues ont besoin de plus de catégories de pluriel CLDR que ta source n'en fournit. L'anglais en a deux (`one` et `other`), tandis que le polonais, l'arabe et le russe en ont davantage. Par défaut, verbatra traduit les formes plurielles présentes dans la source et ne synthétise pas les catégories supplémentaires : l'exécution réussit quand même et émet un avertissement de catégorie de pluriel par locale (code de notification `PLURAL_CATEGORIES_INCOMPLETE`) nommant la locale et les catégories manquantes.

Pour que verbatra synthétise plutôt les formes manquantes, définis `generatePlurals: true` dans ta configuration (ou passe une option `generatePlurals` par exécution). La génération est prise en charge pour les projets i18next-JSON traduits par un fournisseur LLM uniquement ; DeepL, les autres formats et les langues cibles inconnues retombent sur l'avertissement et n'échouent jamais. Les formes générées sont vérifiées pour leur intégrité comme toute traduction et signalées séparément des clés traduites dans le résumé de l'exécution.

## Maintenir la locale source

Tu ne maintiens la locale source qu'à la main. Pour ajouter de nouvelles chaînes, ajoute-les au fichier source et exécute `translate` à nouveau ; verbatra les remplit dans chaque cible, y compris celle que tu viens d'ajouter.
