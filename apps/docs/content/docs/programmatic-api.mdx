The [SDK reference](/docs/sdk) covers every function and type. This page is the hands-on companion: a handful of end-to-end patterns for wiring verbatra into code. Reach for the SDK when you want translation inside a script, a build step, or a CI job, with full control over everything around it.

## Translate in a script

Load the config, run a translation, print the headline. Whole-run problems throw an `SdkError`; per-locale outcomes are data on the summary.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig();
const summary = await translate({ config });

console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Gate a build or CI step on the result

The summary tells you what changed and what failed, no output parsing required. Fail the step when any locale failed, and dig into the per-locale detail to log why.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const summary = await translate({ config: await loadConfig() });

for (const locale of summary.locales) {
  if (locale.status === "failed") {
    console.error(`${locale.locale}: ${locale.error?.code} ${locale.error?.message}`);
  }
}

if (summary.failed.length > 0) {
  process.exitCode = 1;
}
```

## Check without writing

`check` reads and diffs without calling a provider or writing files, and reports per-locale drift split into missing, stale, and up-to-date counts. Its `inSync` flag is true exactly when nothing is missing or stale, which makes it your natural CI gate.

```ts
import { check, loadConfig } from "@verbatra/sdk";

const summary = await check({ config: await loadConfig() });

if (!summary.inSync) {
  const drifted = summary.locales.filter((locale) => !locale.inSync).map((locale) => locale.locale);
  console.error(`Out of sync: ${drifted.join(", ")}. Run verbatra translate.`);
  process.exitCode = 1;
}
```

Want the full run summary (the keys that would be translated) instead of just the drift counts? Use `translate({ dryRun: true })`.

`diff` is the read-only sibling of `check`: same computation, but it returns the actual key lists (`missing`, `changed`, `orphaned`) per locale instead of counts, with the same `hasPendingChanges` gate. Reach for it when you need to know which keys, not just how many.

```ts
import { diff, loadConfig } from "@verbatra/sdk";

const summary = await diff({ config: await loadConfig() });

if (summary.hasPendingChanges) {
  for (const locale of summary.locales) {
    if (locale.hasPendingChanges) {
      console.log(`${locale.locale}: add ${locale.missing.join(", ")}; re-translate ${locale.changed.join(", ")}`);
    }
  }
  process.exitCode = 1;
}
```

## Hand strings to a human translator

Export the strings that need translating into an Excel workbook, then import the filled file back. `importWorkbook` returns the same `RunSummary` as `translate` and runs the same placeholder and ICU checks, so a manual handoff slots into the exact reporting and gating you use for an automated run. See [Human translation](/docs/manual-translation).

```ts
import { exportWorkbook, importWorkbook, loadConfig } from "@verbatra/sdk";

const config = await loadConfig();

// Write a workbook of the strings that need translating.
const exported = await exportWorkbook({ config });
console.log(`Wrote ${exported.path}`);

// ...later, after a translator fills and returns the file, import it back.
const summary = await importWorkbook({ config, workbook: exported.path });
console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Re-translate as the source changes

`watch` fires your `onRun` once per debounced change and returns a controller you stop cleanly.

```ts
import { loadConfig, watch } from "@verbatra/sdk";

const controller = await watch({
  config: await loadConfig(),
  onRun: (result) => {
    if (result.status === "failed") {
      console.error(`run failed: ${result.error.code}`);
    }
  },
});

process.on("SIGINT", () => {
  void controller.stop();
});
```

## A config without a file

`loadConfig` takes an in-memory `configOverride`, so you can drive verbatra entirely from code with no `verbatra.config.ts` on disk.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig({
  configOverride: {
    sourceLocale: "en",
    targetLocales: ["de", "fr"],
    format: "i18next-json",
    files: { pattern: "locales/{locale}.json" },
    provider: { id: "anthropic", options: { model: "claude-sonnet-4-6", maxTokens: 4096 } },
  },
});

await translate({ config });
```
