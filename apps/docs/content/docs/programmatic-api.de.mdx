Die [SDK-Referenz](/docs/sdk) deckt jede Funktion und jeden Typ ab. Diese Seite ist der praktische Begleiter: eine Handvoll End-to-End-Muster, um verbatra in Code einzubinden. Greif zum SDK, wenn du Übersetzung in einem Skript, einem Build-Schritt oder einem CI-Job möchtest, mit voller Kontrolle über alles drumherum.

## In einem Skript übersetzen

Lade die Konfiguration, führe eine Übersetzung aus, gib die Schlagzeile aus. Gesamtlauf-Probleme werfen einen `SdkError`; Ergebnisse pro Locale sind Daten in der Zusammenfassung.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig();
const summary = await translate({ config });

console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Einen Build- oder CI-Schritt am Ergebnis ausrichten

Die Zusammenfassung sagt dir, was sich geändert hat und was fehlgeschlagen ist, kein Ausgabe-Parsen nötig. Lass den Schritt fehlschlagen, wenn eine Locale fehlgeschlagen ist, und grabe dich in das Detail pro Locale, um zu protokollieren, warum.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const summary = await translate({ config: await loadConfig() });

for (const locale of summary.locales) {
  if (locale.status === "failed") {
    console.error(`${locale.locale}: ${locale.error?.code} ${locale.error?.message}`);
  }
}

if (summary.failed.length > 0) {
  process.exitCode = 1;
}
```

## Prüfen ohne zu schreiben

`check` liest und diffe, ohne einen Provider aufzurufen oder Dateien zu schreiben, und meldet die Abweichung pro Locale, aufgeteilt in fehlende, veraltete und aktuelle Anzahlen. Sein `inSync`-Flag ist genau dann true, wenn nichts fehlend oder veraltet ist, was es zu deinem natürlichen CI-Gate macht.

```ts
import { check, loadConfig } from "@verbatra/sdk";

const summary = await check({ config: await loadConfig() });

if (!summary.inSync) {
  const drifted = summary.locales.filter((locale) => !locale.inSync).map((locale) => locale.locale);
  console.error(`Out of sync: ${drifted.join(", ")}. Run verbatra translate.`);
  process.exitCode = 1;
}
```

Willst du die vollständige Lauf-Zusammenfassung (die Keys, die übersetzt würden) statt nur der Abweichungs-Anzahlen? Nutze `translate({ dryRun: true })`.

`diff` ist das schreibgeschützte Geschwister von `check`: dieselbe Berechnung, aber es gibt die tatsächlichen Key-Listen (`missing`, `changed`, `orphaned`) pro Locale statt Anzahlen zurück, mit demselben `hasPendingChanges`-Gate. Greif dazu, wenn du wissen musst, welche Keys, nicht nur wie viele.

```ts
import { diff, loadConfig } from "@verbatra/sdk";

const summary = await diff({ config: await loadConfig() });

if (summary.hasPendingChanges) {
  for (const locale of summary.locales) {
    if (locale.hasPendingChanges) {
      console.log(`${locale.locale}: add ${locale.missing.join(", ")}; re-translate ${locale.changed.join(", ")}`);
    }
  }
  process.exitCode = 1;
}
```

## Strings an einen menschlichen Übersetzer übergeben

Exportiere die Strings, die übersetzt werden müssen, in eine Excel-Arbeitsmappe, dann importiere die ausgefüllte Datei zurück. `importWorkbook` gibt dieselbe `RunSummary` zurück wie `translate` und führt dieselben Platzhalter- und ICU-Prüfungen aus, sodass eine manuelle Übergabe genau in das Reporting und Gating passt, das du für einen automatisierten Lauf nutzt. Siehe [Menschliche Übersetzung](/docs/manual-translation).

```ts
import { exportWorkbook, importWorkbook, loadConfig } from "@verbatra/sdk";

const config = await loadConfig();

// Write a workbook of the strings that need translating.
const exported = await exportWorkbook({ config });
console.log(`Wrote ${exported.path}`);

// ...later, after a translator fills and returns the file, import it back.
const summary = await importWorkbook({ config, workbook: exported.path });
console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Neu übersetzen, während sich die Quelle ändert

`watch` feuert dein `onRun` einmal pro entprellter Änderung und gibt einen Controller zurück, den du sauber stoppst.

```ts
import { loadConfig, watch } from "@verbatra/sdk";

const controller = await watch({
  config: await loadConfig(),
  onRun: (result) => {
    if (result.status === "failed") {
      console.error(`run failed: ${result.error.code}`);
    }
  },
});

process.on("SIGINT", () => {
  void controller.stop();
});
```

## Eine Konfiguration ohne Datei

`loadConfig` nimmt ein In-Memory-`configOverride`, sodass du verbatra vollständig aus dem Code steuern kannst, ganz ohne `verbatra.config.ts` auf der Festplatte.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig({
  configOverride: {
    sourceLocale: "en",
    targetLocales: ["de", "fr"],
    format: "i18next-json",
    files: { pattern: "locales/{locale}.json" },
    provider: { id: "anthropic", options: { model: "claude-sonnet-4-6", maxTokens: 4096 } },
  },
});

await translate({ config });
```

Für die vollständige Liste der Funktionen, die `RunSummary`- und `LocaleSummary`-Strukturen und jeden `SdkError`-Code siehe die [SDK-Referenz](/docs/sdk). Läufst du stattdessen die CLI in CI? Siehe den Exit-Code-Vertrag in [CI and exit codes](/docs/ci-and-exit-codes).
