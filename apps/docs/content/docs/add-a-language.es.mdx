# Añade un idioma

Añadir un idioma es un cambio de una línea en la configuración más una ejecución de traducción. verbatra ve el nuevo locale como si le faltara todo, así que la primera ejecución lo completa por entero, y cada ejecución posterior sigue siendo incremental.

## Añade el locale a tu configuración

Añádelo a `targetLocales` en `verbatra.config.ts`. Lo único que no puede ser es tu `sourceLocale`:

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"], // "fr" added
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "gemini",
    options: {
      model: "gemini-2.5-flash",
      maxOutputTokens: 4096,
    },
  },
});
```

## Traduce

Ejecútalo:

```bash
verbatra translate
```

El nuevo locale todavía no tiene archivo de destino, así que verbatra lo trata como vacío y traduce en él cada clave de origen, escribiendo el archivo a partir de `files.pattern` (aquí `locales/fr.json`). Tus locales existentes quedan intactos a menos que sus cadenas de origen hayan cambiado, porque el archivo de bloqueo sigue teniéndolos registrados como vigentes.

¿Quieres mirar antes de saltar? Previsualiza primero las claves del nuevo locale:

```bash
verbatra translate --dry-run
```

## Idiomas con más categorías de plural

Algunos idiomas necesitan más categorías de plural CLDR que las que tiene tu origen. El inglés tiene dos (`one` y `other`); el polaco, el árabe y el ruso tienen más. Por defecto verbatra traduce las formas de plural que encuentra en el origen y no inventa las adicionales: la ejecución igualmente tiene éxito y emite una advertencia de categoría de plural por locale (código de aviso `PLURAL_CATEGORIES_INCOMPLETE`) que nombra el locale y las categorías que faltan.

Para que verbatra sintetice en su lugar las formas que faltan, define `generatePlurals: true` en tu configuración (o pasa una anulación `generatePlurals` por ejecución):

```ts
export default defineConfig({
  // ...
  generatePlurals: true,
});
```

La generación solo se admite en proyectos i18next-JSON traducidos por un proveedor LLM. DeepL, los demás formatos y los idiomas de destino desconocidos recurren a la advertencia y nunca fallan. Las formas generadas se verifican en integridad como cualquier traducción y se informan por separado de las claves traducidas en el resumen de la ejecución.

## Mantenimiento del locale de origen

Solo mantienes a mano el locale de origen. Para añadir cadenas, ponlas en el archivo de origen y ejecuta `translate` de nuevo: verbatra las rellena en cada destino, incluido el que acabas de añadir.
