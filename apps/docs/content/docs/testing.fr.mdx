# Tests et qualité

verbatra est conçu pour qu'on puisse compter dessus. Chaque changement passe par une suite de tests en couches, et les résultats sont publics pour que tu puisses les vérifier avant de mettre à jour.

## Ce que signifient les badges

- **CI** : le lint, les vérifications de types, les tests unitaires et le build de production s'exécutent sur chaque pull request et sur `main`, sur deux versions de Node. CI impose 90 pour cent de couverture sur les lignes, les fonctions, les instructions et les branches.
- **E2E** : les tarballs publiés de `@verbatra/cli` et `@verbatra/sdk` sont installés dans un projet jetable et pilotés via le vrai binaire, de sorte que les régressions d'empaquetage et de bundling sont détectées avant la publication.
- **Coverage** : la couverture la plus récente rapportée à Codecov depuis l'exécution de CI sur `main`.

## Comment c'est testé

- **Tests unitaires** (Vitest) couvrent la logique métier, les adaptateurs de format, les fournisseurs et le CLI dans chaque paquet.
- **Tests end-to-end** installent les vrais tarballs et exercent toute la surface de commandes : `init`, `check`, `diff`, `export`, `import`, `translate` et `watch`. Un niveau sans clé s'exécute sur chaque pull request ; un niveau en direct traduit contre un vrai fournisseur chaque nuit et après chaque merge.
