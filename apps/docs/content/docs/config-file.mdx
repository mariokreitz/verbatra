# Config file

verbatra reads a small, non-secret configuration object and validates it before doing any work. It carries no API key; the provider reads its key from the environment.

<Callout title="Keys come from the environment">
No API key is ever read from this file. Each provider reads its key from an environment variable, so the config stays safe to commit. See [Providers](/docs/providers).
</Callout>

## A minimal config

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"],
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "anthropic",
    options: {
      model: "claude-sonnet-4-6", // replace with your provider's model id
      maxTokens: 4096,
    },
  },
});
```

`defineConfig` is an identity helper from `@verbatra/sdk`. It returns its argument unchanged and exists only to give you type inference and editor autocomplete.

<Callout title="Model autocomplete in TypeScript">
When you author the config in TypeScript, your editor offers the selected provider's known model IDs for `options.model`, narrowed by the provider `id` you chose. The list comes from that provider's own SDK (`@anthropic-ai/sdk`, `openai`, `@google/genai`), so it stays current with the SDK version you have installed. The field is restricted to that provider's models: a model from another provider (for example a Claude model under `id: "gemini"`) is a type error. The runtime validates `model` only as a non-empty string, so a brand-new model the SDK has not listed yet still works at runtime even though the editor flags it.
</Callout>

## Fields

- **`sourceLocale`** (string): the locale your source strings are written in.
- **`targetLocales`** (string array): the locales to translate into. It must not include `sourceLocale`.
- **`format`** (string): one of `i18next-json`, `vue-i18n-json`, `next-intl-json`, `ngx-translate-json`, `arb`, `yaml`, or `xliff`. See [Formats](/docs/formats) for what each one detects and how it handles placeholders.
- **`files.pattern`** (string): the path to each locale file. It must contain the `{locale}` token, which verbatra replaces with each locale (for example, `locales/{locale}.json` resolves to `locales/de.json`).
- **`provider`** (object): the translation provider, selected by `id`. Each provider takes its own `options`, and in TypeScript the `model` field is restricted to the chosen provider's known model IDs (a model from another provider is a type error). See [Providers](/docs/providers).
- **`glossary`** (optional, object): a map of source terms to preferred target terms. The LLM providers (Anthropic, OpenAI, Gemini) receive it with the request and are instructed to treat its terms as binding. DeepL does not apply this term map; it records a notice and instead applies a glossary configured by id (see [Providers](/docs/providers)).
- **`tone`** (optional): one of `"formal"`, `"informal"`, or `"neutral"`. The LLM providers receive it and are instructed to honor it. DeepL maps it to formality: `formal` becomes more formal, `informal` becomes less formal, and `neutral` or an absent tone is omitted. On a free-tier DeepL key, a non-default tone is not supported and degrades to the default with a notice.
- **`prune`** (optional, boolean, default `false`): when true, keys present in a target file but absent from the source (the diff's orphaned keys) are removed from the written file and the lock. A per-run `prune` option on `translate` (the CLI `--prune` flag) overrides this. Only orphaned keys are ever removed.
- **`generatePlurals`** (optional, boolean, default `false`): when true, verbatra synthesizes the CLDR plural forms a target language requires but the source does not supply (for example Polish few/many). Supported for i18next-JSON projects translated by an LLM provider only; DeepL, non-i18next formats, and unknown target languages fall back to the per-locale plural warning. Unlike `prune`, there is no `--generate-plurals` CLI flag, so set this in the config; the SDK `translate()` input does accept a per-run `generatePlurals` override that takes precedence.
- **`maxBatchSize`** (optional, positive integer, default `50`): a locale's missing-plus-changed entries are split into sequential sub-batches no larger than this, so one oversized provider request cannot fail the whole locale. A failed sub-batch is withheld and retried next run while the others still make progress. This is config-only: there is no CLI flag and no per-run `translate()` override, so set it in the config.

Unknown top-level keys are rejected. This is also why an API key cannot live in the config: keys come from the environment, not the file.

A config that uses the optional fields:

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"],
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  glossary: {
    "Sign in": "Anmelden",
  },
  tone: "formal",
  maxBatchSize: 25,
  provider: {
    id: "anthropic",
    options: {
      model: "claude-sonnet-4-6",
      maxTokens: 4096,
    },
  },
});
```

## Discovery order

verbatra searches upward from the working directory and uses the first source it finds. Multiple present sources are not an error; the first match wins. The search order is:

1. `package.json` (a `"verbatra"` property)
2. `.verbatrarc`
3. `.verbatrarc.json`
4. `.verbatrarc.yaml`
5. `.verbatrarc.yml`
6. `.verbatrarc.js`
7. `.verbatrarc.cjs`
8. `.verbatrarc.ts`
9. `verbatra.config.js`
10. `verbatra.config.cjs`
11. `verbatra.config.ts`

To load one explicit file instead of searching, pass `--config <path>`.

## The working directory and .env

Run verbatra from your project root, or pass `--cwd` pointing at it. The working directory is the single anchor for everything verbatra resolves: the config search starts there and walks upward, `.env.local` and `.env` are loaded from there, and each locale file path is resolved against it. Keeping all three in the same place is what makes a plain `verbatra translate` work.

Environment precedence is, highest first: a variable already set in your real environment, then `.env.local`, then `.env`. A variable already present is never overwritten.
