# API programática

La [referencia del SDK](/docs/sdk) documenta cada función y tipo. Esta página es el complemento orientado a tareas: unos pocos patrones de extremo a extremo para usar verbatra en código. Recurre al SDK cuando quieras traducción dentro de un script, un paso de compilación o un trabajo de CI, con control total sobre lo que ocurre a su alrededor.

## Traducir en un script

Carga la configuración, ejecuta una traducción e informa el titular. Los problemas de toda la ejecución lanzan un `SdkError`; los resultados por configuración regional son datos en el resumen.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig();
const summary = await translate({ config });

console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Condicionar una compilación o un paso de CI según el resultado

El resumen te dice qué cambió y qué falló sin analizar la salida. Haz fallar el paso cuando alguna configuración regional falle, e inspecciona el detalle por configuración regional para registrar el motivo.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const summary = await translate({ config: await loadConfig() });

for (const locale of summary.locales) {
  if (locale.status === "failed") {
    console.error(`${locale.locale}: ${locale.error?.code} ${locale.error?.message}`);
  }
}

if (summary.failed.length > 0) {
  process.exitCode = 1;
}
```

## Comprobar sin escribir

`check` lee y hace diff sin llamar a un proveedor ni escribir archivos, e informa la desviación por configuración regional dividida en recuentos de faltantes, desactualizadas y actualizadas. Su bandera `inSync` es true exactamente cuando no hay nada faltante ni desactualizado, por lo que es la puerta de CI natural.

```ts
import { check, loadConfig } from "@verbatra/sdk";

const summary = await check({ config: await loadConfig() });

if (!summary.inSync) {
  const drifted = summary.locales.filter((locale) => !locale.inSync).map((locale) => locale.locale);
  console.error(`Out of sync: ${drifted.join(", ")}. Run verbatra translate.`);
  process.exitCode = 1;
}
```

`translate({ dryRun: true })` es una alternativa cuando quieres el resumen completo de la ejecución (las claves que se traducirían) en lugar de solo los recuentos de desviación.

`diff` es el hermano de solo lectura de `check`: ejecuta el mismo cálculo pero devuelve las listas de claves reales (`missing`, `changed`, `orphaned`) por configuración regional en lugar de recuentos, con la misma puerta `hasPendingChanges`. Recurre a él cuando necesites qué claves, no solo cuántas.

```ts
import { diff, loadConfig } from "@verbatra/sdk";

const summary = await diff({ config: await loadConfig() });

if (summary.hasPendingChanges) {
  for (const locale of summary.locales) {
    if (locale.hasPendingChanges) {
      console.log(`${locale.locale}: add ${locale.missing.join(", ")}; re-translate ${locale.changed.join(", ")}`);
    }
  }
  process.exitCode = 1;
}
```

## Entregar cadenas a un traductor humano

Exporta las cadenas que necesitan traducción a un libro de Excel, y luego importa de vuelta el archivo rellenado. `importWorkbook` devuelve el mismo `RunSummary` que `translate` y ejecuta las mismas comprobaciones de marcadores de posición e ICU, así que una entrega manual encaja en el mismo informe y las mismas puertas que una ejecución automatizada. Consulta [Traducción manual](/docs/manual-translation).

```ts
import { exportWorkbook, importWorkbook, loadConfig } from "@verbatra/sdk";

const config = await loadConfig();

// Write a workbook of the strings that need translating.
const exported = await exportWorkbook({ config });
console.log(`Wrote ${exported.path}`);

// ...later, after a translator fills and returns the file, import it back.
const summary = await importWorkbook({ config, workbook: exported.path });
console.log(`${summary.succeeded.length} ok, ${summary.failed.length} failed`);
```

## Volver a traducir a medida que cambia el origen

`watch` llama a tu `onRun` una vez por cada cambio con rebote y devuelve un controlador que detienes de forma limpia.

```ts
import { loadConfig, watch } from "@verbatra/sdk";

const controller = await watch({
  config: await loadConfig(),
  onRun: (result) => {
    if (result.status === "failed") {
      console.error(`run failed: ${result.error.code}`);
    }
  },
});

process.on("SIGINT", () => {
  void controller.stop();
});
```

## Una configuración sin archivo

`loadConfig` acepta un `configOverride` en memoria, así que puedes manejar verbatra enteramente desde código sin un `verbatra.config.ts` en disco.

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig({
  configOverride: {
    sourceLocale: "en",
    targetLocales: ["de", "fr"],
    format: "i18next-json",
    files: { pattern: "locales/{locale}.json" },
    provider: { id: "anthropic", options: { model: "claude-sonnet-4-6", maxTokens: 4096 } },
  },
});

await translate({ config });
```

Para la lista completa de funciones, las formas de `RunSummary` y `LocaleSummary`, y cada código de `SdkError`, consulta la [referencia del SDK](/docs/sdk). Para el contrato del código de salida cuando ejecutas la CLI en CI en su lugar, consulta [CI y códigos de salida](/docs/ci-and-exit-codes).
