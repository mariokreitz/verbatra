# Archivo de configuración

verbatra lee un pequeño objeto de configuración no secreto y lo valida antes de realizar cualquier trabajo. No contiene ninguna clave de API; el proveedor lee su clave del entorno.

<Callout title="Keys come from the environment">
Nunca se lee ninguna clave de API de este archivo. Cada proveedor lee su clave de una variable de entorno, por lo que la configuración permanece segura para ser confirmada. Consulta [Proveedores](/docs/providers).
</Callout>

## Una configuración mínima

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"],
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "anthropic",
    options: {
      model: "claude-sonnet-4-6", // replace with your provider's model id
      maxTokens: 4096,
    },
  },
});
```

`defineConfig` es una función auxiliar de identidad de `@verbatra/sdk`. Devuelve su argumento sin cambios y existe solo para proporcionarte inferencia de tipos y autocompletado en el editor.

<Callout title="Autocompletado de modelos en TypeScript">
Cuando escribes la configuración en TypeScript, tu editor ofrece para `options.model` los IDs de modelo conocidos del proveedor seleccionado, acotados por la `id` de proveedor que elegiste. La lista proviene del propio SDK de ese proveedor (`@anthropic-ai/sdk`, `openai`, `@google/genai`), por lo que se mantiene actualizada con la versión del SDK que tengas instalada. El campo está restringido a los modelos de ese proveedor: un modelo de otro proveedor (por ejemplo, un modelo de Claude con `id: "gemini"`) es un error de tipo. En tiempo de ejecución, `model` solo se valida como una cadena no vacía, por lo que un modelo recién lanzado que el SDK aún no liste funciona en tiempo de ejecución aunque el editor lo marque.
</Callout>

## Campos

- **`sourceLocale`** (string): la configuración regional en la que están escritas tus cadenas de origen.
- **`targetLocales`** (array de strings): las configuraciones regionales a las que traducir. No debe incluir `sourceLocale`.
- **`format`** (string): uno de `i18next-json`, `vue-i18n-json`, `next-intl-json`, o `ngx-translate-json`.
- **`files.pattern`** (string): la ruta a cada archivo de configuración regional. Debe contener el token `{locale}`, que verbatra reemplaza con cada configuración regional (por ejemplo, `locales/{locale}.json` se resuelve en `locales/de.json`).
- **`provider`** (objeto): el proveedor de traducción, seleccionado por `id`. Cada proveedor tiene sus propias `options`, y en TypeScript el campo `model` está restringido a los IDs de modelo conocidos del proveedor elegido (un modelo de otro proveedor es un error de tipo). Consulta [Proveedores](/docs/providers).
- **`glossary`** (opcional, objeto): un mapa de términos de origen a términos de destino preferidos. Los proveedores de LLM (Anthropic, OpenAI, Gemini) lo reciben con la solicitud y se les indica que traten sus términos como vinculantes. DeepL no aplica este mapa de términos; registra un aviso y, en su lugar, aplica un glosario configurado por id (consulta [Proveedores](/docs/providers)).
- **`tone`** (opcional): uno de `"formal"`, `"informal"`, o `"neutral"`. Los proveedores de LLM lo reciben y se les indica que lo respeten. DeepL lo mapea a la formalidad: `formal` se vuelve más formal, `informal` se vuelve menos formal, y `neutral` o un tono ausente se omite. Con una clave DeepL de nivel gratuito, un tono no predeterminado no es compatible y se degrada al predeterminado con un aviso.
- **`prune`** (opcional, booleano, por defecto `false`): cuando es true, las claves presentes en un archivo de destino pero ausentes del origen (las claves huérfanas del diff) se eliminan del archivo escrito y del bloqueo. Una opción `prune` por ejecución en `translate` (la bandera `--prune` de la CLI) la anula. Solo las claves huérfanas se eliminan en algún caso.
- **`generatePlurals`** (opcional, booleano, por defecto `false`): cuando es true, verbatra sintetiza las formas plurales CLDR que una lengua de destino requiere pero el origen no aporta (por ejemplo, few/many en polaco). Compatible solo con proyectos i18next-JSON traducidos por un proveedor de LLM; DeepL, los formatos no i18next y las lenguas de destino desconocidas recurren a la advertencia plural por configuración regional. A diferencia de `prune`, no hay un flag de CLI `--generate-plurals`, así que configúralo en la configuración; la entrada `translate()` del SDK sí acepta una opción `generatePlurals` por ejecución que tiene prioridad.

Las claves de nivel superior desconocidas son rechazadas. Esta es también la razón por la que una clave de API no puede residir en la configuración: las claves provienen del entorno, no del archivo.

Una configuración que utiliza los campos opcionales:

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"],
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  glossary: {
    "Sign in": "Anmelden",
  },
  tone: "formal",
  provider: {
    id: "anthropic",
    options: {
      model: "claude-sonnet-4-6",
      maxTokens: 4096,
    },
  },
});
```

## Orden de descubrimiento

verbatra busca hacia arriba desde el directorio de trabajo y utiliza la primera fuente que encuentra. Múltiples fuentes presentes no son un error; la primera coincidencia gana. El orden de búsqueda es:

1. `package.json` (una propiedad `"verbatra"`)
2. `.verbatrarc`
3. `.verbatrarc.json`
4. `.verbatrarc.yaml`
5. `.verbatrarc.yml`
6. `.verbatrarc.js`
7. `.verbatrarc.cjs`
8. `.verbatrarc.ts`
9. `verbatra.config.js`
10. `verbatra.config.cjs`
11. `verbatra.config.ts`

Para cargar un archivo explícito en lugar de buscar, pasa `--config <path>`.

## El directorio de trabajo y .env

Ejecuta verbatra desde la raíz de tu proyecto, o pasa `--cwd` apuntando a ella. El directorio de trabajo es el único ancla para todo lo que verbatra resuelve: la búsqueda de configuración comienza allí y sube, `.env.local` y `.env` se cargan desde allí, y cada ruta de archivo de configuración regional se resuelve en relación con él. Mantener los tres en el mismo lugar es lo que hace que un simple `verbatra translate` funcione.

La precedencia del entorno es, de mayor a menor: una variable ya establecida en tu entorno real, luego `.env.local`, luego `.env`. Una variable ya presente nunca se sobrescribe.
