# SDK

`@verbatra/sdk` is the programmatic core that the `verbatra` command is built on. The CLI is a thin
wrapper over it, so anything the command line does, you can do in code: load and validate a config,
run a one-shot translation, or watch the source and re-translate on each change. Reach for the SDK
when you want translation inside a script, a CI job, or your own tooling.

## Install

```bash
pnpm add -D @verbatra/sdk
# npm
npm install -D @verbatra/sdk
# yarn
yarn add -D @verbatra/sdk
```

Requires Node.js `>=22.14.0`.

## The public surface

The package exports eight functions and the types a caller touches.

- `defineConfig(config): VerbatraConfig` returns its argument unchanged. It exists only to give you
  type inference and autocomplete when you author a config in TypeScript; the `model` field is
  restricted to the selected provider's known models, so another provider's model is a type error.
- `loadConfig(options?: LoadConfigOptions): Promise<VerbatraConfig>` discovers and validates the
  config. `LoadConfigOptions` accepts `cwd`, an explicit `configPath`, or an in-memory
  `configOverride`. It throws an `SdkError` when no config is found or validation fails.
- `translate(input: TranslateInput): Promise<RunSummary>` runs the read, diff, translate, write flow
  over every target locale. `TranslateInput` is `{ config, cwd?, dryRun?, prune?, generatePlurals? }`.
  `prune` (default off) removes orphaned keys from the written file and the lock; `generatePlurals`
  (default off) synthesizes the CLDR plural forms a target language needs but the source lacks
  (i18next-JSON with an LLM provider only). Each per-run flag, when set, overrides the matching config
  option for that run. It resolves to a `RunSummary`; whole-run problems throw an `SdkError`, while
  per-locale outcomes are data on the summary.
- `watch(input: WatchInput): Promise<WatchController>` watches the source file and re-runs on each
  debounced change. `WatchInput` is `{ config, cwd?, debounceMs?, onRun }`, where `onRun` receives a
  `WatchRunResult` per run. The returned `WatchController` has a single method, `stop()`, which closes
  the watcher and awaits the in-flight run.
- `exportWorkbook(input: ExportWorkbookInput): Promise<ExportWorkbookResult>` writes the strings that
  need translating into a styled Excel workbook for a human translator. `ExportWorkbookInput` is
  `{ config, cwd?, out?, locales?, includeUnchanged? }`. It resolves to an `ExportWorkbookResult`:
  the absolute `path` written and the per-locale `rows` counts. No provider is called and no lock
  file is written. See [Manual translation](/docs/manual-translation).
- `importWorkbook(input: ImportWorkbookInput): Promise<RunSummary>` reads a filled workbook back into
  the locale files, running the same placeholder and ICU checks as `translate`. `ImportWorkbookInput`
  is `{ config, workbook, cwd?, dryRun? }`. It resolves to the same `RunSummary` as `translate`, so a
  per-locale failure is data on the summary rather than a throw. `DEFAULT_WORKBOOK_PATH` is the
  default export path (`verbatra-translations.xlsx`).
- `check(input: CheckInput): Promise<CheckSummary>` runs the same read and diff as `translate` but
  stops before any provider call: it reports per-locale drift and writes nothing. `CheckInput` is
  `{ config, cwd?, locales? }`. It resolves to a `CheckSummary` whose `inSync` is true exactly when
  every checked locale has no missing and no stale keys. No provider is called, no file is written,
  and the lock is not touched. See [`verbatra check`](/docs/cli/check). For the keys behind those
  counts, use `diff` below.
- `diff(input: DiffInput): Promise<DiffSummary>` is the read-only sibling of `check`: it runs the
  same read and diff but reports key lists instead of counts. `DiffInput` is `{ config, cwd?, locales? }`.
  It resolves to a `DiffSummary` whose per-locale entries carry `missing` (keys that would be added),
  `changed` (keys that would be re-translated), and `orphaned` (keys in the target but not the source,
  reported only). Its `hasPendingChanges` is true exactly when some locale has a missing or changed key;
  orphaned keys alone do not set it. No provider is called, no file is written, and the lock is not
  touched. See [`verbatra diff`](/docs/cli/diff).

The result types:

```ts
interface RunSummary {
  dryRun: boolean;          // true when nothing was written and no provider was called
  locales: LocaleSummary[]; // one entry per target locale, in config order
  succeeded: string[];      // locales whose run succeeded
  failed: string[];         // locales whose run failed
}

interface LocaleSummary {
  locale: string;
  status: "succeeded" | "failed";
  translated: string[];          // keys translated this run (in dry-run, keys that would be)
  unchanged: string[];           // keys already up to date
  orphaned: string[];            // target keys with no matching source key (reported regardless of pruning)
  pruned: string[];              // orphaned keys removed this run; empty unless pruning is on (subset of orphaned)
  invalidIcuSource: string[];    // source keys skipped for invalid ICU
  integrityMismatches: string[]; // translations withheld for a placeholder mismatch
  generated: string[];           // CLDR plural forms synthesized this run; empty unless generation is on
  notices: LocaleNotice[];       // provider notices (e.g. DeepL degradation) and SDK notices (e.g. PLURAL_CATEGORIES_INCOMPLETE)
  error?: { code: string; message: string }; // present only when status is "failed"
}

// A notice is either a provider-emitted notice or an SDK-emitted one.
type LocaleNotice = ProviderNotice | SdkNotice;

type WatchRunResult =
  | { status: "succeeded"; summary: RunSummary }
  | { status: "failed"; error: { code: string; message: string } };

interface CheckSummary {
  inSync: boolean;                 // true exactly when every locale is in sync
  locales: LocaleCheckSummary[];   // one entry per checked locale, in config order
}

interface LocaleCheckSummary {
  locale: string;
  missing: number;                 // in source, absent from target
  stale: number;                   // source changed since last translated
  upToDate: number;                // target matches the current source
  inSync: boolean;                 // missing === 0 && stale === 0
}

interface DiffSummary {
  hasPendingChanges: boolean;      // true when some locale has a missing or changed key
  locales: LocaleDiff[];           // one entry per checked locale, in config order
}

interface LocaleDiff {
  locale: string;
  missing: string[];               // in source, absent from target: would be added
  changed: string[];               // source changed since last translated: would be re-translated
  orphaned: string[];              // in target, absent from source: reported only
  hasPendingChanges: boolean;      // missing.length > 0 || changed.length > 0
}
```

Notices are per locale, on `LocaleSummary.notices`, and cover both provider notices (for example a
DeepL graceful degradation) and SDK notices (for example a target language needing more CLDR plural
categories than the source supplies, code `PLURAL_CATEGORIES_INCOMPLETE`). There is no top-level
notices list on `RunSummary`.

`SdkError` is the single error type the SDK throws for whole-run failures. It carries a stable `code`
and a secret-free `message`. The codes are:

| Code | When |
| --- | --- |
| `CONFIG_NOT_FOUND` | no config was found, or an explicit `configPath` does not exist |
| `CONFIG_INVALID` | a config was found but is unparseable or fails validation |
| `UNKNOWN_FORMAT` | no adapter is registered for the configured format |
| `PROVIDER_CONSTRUCTION_FAILED` | the provider could not be built (includes a missing API key) |
| `SOURCE_UNREADABLE` | the source locale file does not exist |
| `SOURCE_INVALID` | the source locale file could not be read or parsed |
| `LOCK_FILE_INVALID` | the lock file is present but corrupt or too large |
| `LOCALE_FAILED` | not thrown: the fallback `code` recorded on a failed locale's `error` |

A single locale failing does not throw; it is recorded on that locale's `LocaleSummary` with `status:
"failed"` and an `error`, and the run continues.

## Define a config

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"],
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "anthropic",
    options: {
      model: "claude-sonnet-4-6",
      maxTokens: 4096,
    },
  },
});
```

## Run a one-shot translation

`translate` resolves to the run summary. Read `succeeded` and `failed` for the headline, and walk
`locales` for the detail, including any notices (provider or SDK).

```ts
import { loadConfig, translate } from "@verbatra/sdk";

const config = await loadConfig();
const summary = await translate({ config });

console.log(`${summary.succeeded.length} succeeded, ${summary.failed.length} failed`);

for (const locale of summary.locales) {
  for (const notice of locale.notices) {
    console.warn(`${locale.locale}: ${notice.code} ${notice.message}`);
  }
}
```

Pass `dryRun: true` to read and diff without calling a provider or writing files. Pass
`prune: true` to remove orphaned keys, or `generatePlurals: true` to synthesize missing plural
categories (i18next-JSON with an LLM provider only); each overrides the matching config option for
that run. Unlike `prune` and `generatePlurals`, `maxBatchSize` is config-only (set on the config
object, default 50): there is no per-run `translate()` override for it.

## Watch mode

`watch` calls your `onRun` once per run with a discriminated `WatchRunResult`. The controller's
`stop()` shuts it down cleanly.

```ts
import { loadConfig, watch } from "@verbatra/sdk";

const config = await loadConfig();
const controller = await watch({
  config,
  onRun: (result) => {
    if (result.status === "succeeded") {
      console.log(`${result.summary.succeeded.length} ok, ${result.summary.failed.length} failed`);
    } else {
      console.error(`run failed: ${result.error.code} ${result.error.message}`);
    }
  },
});

// Stop cleanly on Ctrl-C.
process.on("SIGINT", () => {
  void controller.stop();
});
```

## Handle errors

Whole-run failures throw an `SdkError`. Switch on its `code` to react to specific cases.

```ts
import { loadConfig, SdkError, translate } from "@verbatra/sdk";

try {
  const config = await loadConfig();
  await translate({ config });
} catch (error) {
  if (error instanceof SdkError) {
    switch (error.code) {
      case "CONFIG_NOT_FOUND":
        console.error("No verbatra config found. Run `verbatra init` to scaffold one.");
        break;
      case "SOURCE_UNREADABLE":
      case "SOURCE_INVALID":
        console.error(`Could not read the source locale: ${error.message}`);
        break;
      default:
        console.error(`${error.code}: ${error.message}`);
    }
  } else {
    throw error;
  }
}
```
