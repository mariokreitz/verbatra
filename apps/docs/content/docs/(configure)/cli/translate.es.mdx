---
title: verbatra translate
description: Traduce cada locale de destino una vez y luego sale.
---
Ejecuta el flujo de leer, comparar, traducir y escribir sobre cada locale de destino una vez, y luego sale. verbatra lee el locale de origen, lo compara con lo que tradujo la última vez y envía al proveedor solo las cadenas nuevas o modificadas. Todo lo que sigue vigente queda intacto, así que nunca pagas por volver a traducir lo que no se ha movido.

```bash
verbatra translate
```

## Opciones

| Opción | Descripción |
| --- | --- |
| `--cwd <path>` | resuelve la configuración y los archivos de locale desde este directorio |
| `--config <path>` | carga este archivo de configuración en lugar de buscar uno |
| `--dry-run` | previsualiza los cambios sin llamar a un proveedor ni escribir archivos |
| `--prune` | elimina las claves huérfanas (en un archivo de destino pero ausentes del origen) del archivo escrito y del bloqueo |
| `--json` | imprime el resumen de la ejecución como JSON |

Por defecto, una ejecución no elimina nada. Las claves huérfanas (presentes en un archivo de destino pero ya no en el origen) se informan y se dejan en su lugar. Pasa `--prune` para eliminar exactamente esas claves del archivo escrito y del bloqueo, y nada más. Este flag por ejecución anula la opción `prune` de tu [configuración](/docs/config-file); combínalo con `--dry-run` para previsualizar las claves que se irían. ¿Quieres ver qué claves están huérfanas antes de una ejecución? Usa [`verbatra diff`](/docs/cli/diff).

## Salida

Sin `--json`, `translate` imprime un resumen de la ejecución legible para humanos. Con `--json`, imprime el resumen de la ejecución como un único objeto JSON en stdout: consulta la [forma de `RunSummary` en la referencia del SDK](/docs/sdk). Un error de toda la ejecución va a stderr, así que stdout queda limpio y canalizable con `--json`.

Una línea de locale muestra un contador `needs-review` solo cuando es mayor que cero: las propias heurísticas de verbatra que marcan una clave traducida digna de una segunda mirada (una salida sospechosamente corta o larga, un valor idéntico al origen, un término de glosario que se perdió, o un conjunto de marcadores de posición reordenado). No retiene nada por sí mismo; consulta [Seguridad de la traducción](/docs/translation-safety) para saber qué significa cada motivo. La salida `--json` siempre lleva el detalle completo como `needsReview` en cada locale.

## Ejemplos

```bash
# translate once using the config it finds
verbatra translate

# preview changes without calling a provider or writing files
verbatra translate --dry-run

# also remove orphaned keys from target files
verbatra translate --prune

# preview the keys that would be pruned
verbatra translate --prune --dry-run

# run against a project in another directory
verbatra translate --cwd packages/app

# load a specific config instead of searching for one
verbatra translate --config config/verbatra.config.ts

# machine-readable summary on stdout
verbatra translate --json
```

## Códigos de salida

| Código | Significado |
| --- | --- |
| `0` | cada locale de destino tuvo éxito |
| `1` | la ejecución terminó pero uno o más locales fallaron |
| `2` | no se pudo ejecutar: un error de toda la ejecución (configuración ausente o no válida, origen ilegible, formato desconocido, un proveedor que no se pudo construir incluyendo una clave de API ausente, o un archivo de bloqueo corrupto), o un error de uso |

Que un solo locale falle no es fatal: produce el código de salida `1`, y el detalle por locale está en el resumen. Consulta [CI y automatización](/docs/ci-and-exit-codes) para bifurcar sobre estos códigos.
