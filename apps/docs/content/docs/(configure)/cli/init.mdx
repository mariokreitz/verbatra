---
title: verbatra init
description: Scaffold a verbatra config and .env example for your project.
---
Scaffold a `verbatra.config.ts` and a `.env.example`, and add `.env`, `.env.local`, and `.verbatra-local/` to your `.gitignore`, so a real key never lands in a commit and the local run data `translate` and `watch` write there (such as `run-status.json`) stays out of it too. With a terminal attached and no `--yes`, `init` prompts for the values it cannot default; otherwise it runs non-interactively and takes the defaults.

Starting a small project? Gemini has a free tier, so it is a good default to scaffold with:

```bash
verbatra init --provider gemini
```

## Options

| Option | Description |
| --- | --- |
| `--cwd <path>` | write the config and env files to this directory |
| `--provider <id>` | translation provider: `anthropic`, `openai`, `gemini`, or `deepl` (required unless prompted) |
| `--source <locale>` | locale your source strings are written in (default `en`) |
| `--targets <locales>` | comma-separated locales to translate into (default `de`) |
| `--path <pattern>` | locale file pattern containing the `{locale}` token (default `locales/{locale}.json`) |
| `--yes` | skip prompts and accept the defaults |
| `--force` | overwrite an existing config or `.env.example` |

The provider is the one value `init` cannot guess. Without a terminal to prompt (in CI, or with `--yes`), pass `--provider`. Everything else falls back to its default when you leave it out.

## Existing files

By default, `init` never overwrites. If a `verbatra.config.ts` or `.env.example` is already there, it stays untouched and the command still succeeds. Pass `--force` when you actually want to overwrite.

## Examples

```bash
# create config + .env example, prompting for the rest
verbatra init --provider gemini

# non-interactive, accept all defaults
verbatra init --provider deepl --yes

# set the source and target locales up front
verbatra init --provider gemini --source en --targets de,es,fr

# point at a non-default locale file layout
verbatra init --provider gemini --path src/locales/{locale}.json

# scaffold into another directory
verbatra init --provider gemini --cwd packages/app

# overwrite an existing config and .env example
verbatra init --provider gemini --force
```

## Exit codes

| Code | Meaning |
| --- | --- |
| `0` | the files were written, or safely skipped because they already exist (without `--force`) |
| `2` | a usage error: no `--provider` was given without a terminal to prompt, or the assembled config failed validation |

Once `init` finishes, set the provider's API key and run [`verbatra translate`](/docs/cli/translate). See [the quick start](/docs) for the full walkthrough.
