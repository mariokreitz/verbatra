---
title: verbatra init
description: Erstellt das Grundgerüst einer verbatra-Konfiguration und einer .env-Beispieldatei für dein Projekt.
---
Erstellt eine `verbatra.config.ts` und eine `.env.example` und fügt `.env` und `.env.local` zu deiner `.gitignore` hinzu, damit ein echter Schlüssel niemals in einem Commit landet. Mit einem angeschlossenen Terminal und ohne `--yes` fragt `init` nach den Werten, für die es keine Standardwerte hat; andernfalls läuft es nicht-interaktiv und verwendet die Standardwerte.

Startest du ein kleines Projekt? Gemini hat eine kostenlose Stufe, daher ist es ein guter Standard für das Grundgerüst:

```bash
verbatra init --provider gemini
```

## Optionen

| Option | Beschreibung |
| --- | --- |
| `--cwd <path>` | schreibt die Konfigurations- und Umgebungsdateien in dieses Verzeichnis |
| `--provider <id>` | Übersetzungs-Provider: `anthropic`, `openai`, `gemini` oder `deepl` (erforderlich, sofern nicht abgefragt) |
| `--source <locale>` | Locale, in dem deine Quell-Strings geschrieben sind (Standard `en`) |
| `--targets <locales>` | kommagetrennte Locales, in die übersetzt werden soll (Standard `de`) |
| `--path <pattern>` | Locale-Dateimuster, das das `{locale}`-Token enthält (Standard `locales/{locale}.json`) |
| `--yes` | überspringt Aufforderungen und akzeptiert die Standardwerte |
| `--force` | überschreibt eine bestehende Konfiguration oder `.env.example` |

Der Provider ist der eine Wert, den `init` nicht erraten kann. Ohne ein Terminal zum Abfragen (in CI oder mit `--yes`) übergib `--provider`. Alles andere fällt auf seinen Standardwert zurück, wenn du es weglässt.

## Bestehende Dateien

Standardmäßig überschreibt `init` niemals. Wenn eine `verbatra.config.ts` oder `.env.example` bereits vorhanden ist, bleibt sie unberührt und der Befehl ist trotzdem erfolgreich. Übergib `--force`, wenn du wirklich überschreiben willst.

## Beispiele

```bash
# create config + .env example, prompting for the rest
verbatra init --provider gemini

# non-interactive, accept all defaults
verbatra init --provider deepl --yes

# set the source and target locales up front
verbatra init --provider gemini --source en --targets de,es,fr

# point at a non-default locale file layout
verbatra init --provider gemini --path src/locales/{locale}.json

# scaffold into another directory
verbatra init --provider gemini --cwd packages/app

# overwrite an existing config and .env example
verbatra init --provider gemini --force
```

## Exit-Codes

| Code | Bedeutung |
| --- | --- |
| `0` | die Dateien wurden geschrieben oder sicher übersprungen, weil sie bereits existieren (ohne `--force`) |
| `2` | ein Nutzungsfehler: ohne ein Terminal zum Abfragen wurde kein `--provider` angegeben, oder die zusammengesetzte Konfiguration ist bei der Validierung fehlgeschlagen |

Sobald `init` fertig ist, setze den API-Schlüssel des Providers und führe [`verbatra translate`](/docs/cli/translate) aus. Siehe [den Schnellstart](/docs) für die vollständige Anleitung.
