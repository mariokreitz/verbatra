---
title: verbatra export
description: Export untranslated strings into a styled Excel workbook for a human translator.
---
Write the strings that need translating into a styled Excel workbook a human can fill in, then send back for [`verbatra import`](/docs/cli/import). The workbook has one sheet per target locale plus an instructions sheet. The translator edits a single column; everything else is locked.

```bash
verbatra export
```

By default this writes `verbatra-translations.xlsx` in the working directory with only the **missing and changed** strings: the same keys an automated `translate` run would send to a provider. See [Human translation](/docs/manual-translation) for the full round-trip.

Every row with a current translation also carries a read-only `Review status` / `Review reasons` pair: verbatra's own advisory heuristics for spotting a translation worth a second look, recomputed fresh from the current source and target values at export time. They never affect what `import` accepts. See [Human translation](/docs/manual-translation) for the columns and [Translation safety](/docs/translation-safety) for the reason codes.

## Options

| Option | Description |
| --- | --- |
| `--cwd <path>` | resolve config and locale files from this directory |
| `--config <path>` | load this config file instead of searching for one |
| `--out <path>` | write the workbook to this path (default `verbatra-translations.xlsx`) |
| `--locales <list>` | comma-separated subset of target locales (default all configured) |
| `--include-unchanged` | also export already up-to-date strings (off by default) |
| `--json` | print the export result as JSON |

## Output

Without `--json`, `export` prints the output path and one line per locale with its row count. With `--json`, it prints the export result (the path written and the per-locale row counts) as one JSON object on stdout. A whole-run error goes to stderr, so stdout stays clean and pipeable under `--json`.

## Examples

```bash
# write the workbook with missing and changed strings
verbatra export

# only the German and French sheets
verbatra export --locales de,fr

# include already up-to-date strings for full context
verbatra export --include-unchanged

# write to a specific path
verbatra export --out handoff/round-2.xlsx

# export a project in another directory
verbatra export --cwd packages/app

# load a specific config instead of searching for one
verbatra export --config config/verbatra.config.ts

# machine-readable result on stdout
verbatra export --json
```

## Exit codes

| Code | Meaning |
| --- | --- |
| `0` | the workbook was written |
| `2` | could not run: a whole-run error (missing or invalid config, unreadable source, unknown format, a corrupt lock file, or a requested locale not among the configured targets), or a usage error (including an empty `--locales` list) |

`export` never calls a provider and needs no API key. An unknown or empty `--locales` value fails the run with exit `2` rather than silently exporting nothing, so a typo in a locale name is caught instead of passing. See [`verbatra import`](/docs/cli/import) for reading the filled workbook back.
