---
title: verbatra init
description: Échafaude une configuration verbatra et un exemple de .env pour ton projet.
---
Crée un `verbatra.config.ts` et un `.env.example`, et ajoute `.env`, `.env.local` et `.verbatra-local/` à ton `.gitignore`, afin qu'une vraie clé ne se retrouve jamais dans un commit et que les données locales d'exécution que `translate` et `watch` y écrivent (comme `run-status.json`) restent elles aussi hors du dépôt. Avec un terminal attaché et sans `--yes`, `init` demande les valeurs qu'il ne peut pas définir par défaut ; sinon, il s'exécute de façon non interactive et prend les valeurs par défaut.

Tu démarres un petit projet ? Gemini a un niveau gratuit, c'est donc une bonne valeur par défaut pour échafauder :

```bash
verbatra init --provider gemini
```

## Options

| Option | Description |
| --- | --- |
| `--cwd <path>` | écrire la configuration et les fichiers env dans ce répertoire |
| `--provider <id>` | fournisseur de traduction : `anthropic`, `openai`, `gemini` ou `deepl` (requis sauf si demandé) |
| `--source <locale>` | locale dans laquelle tes chaînes source sont écrites (par défaut `en`) |
| `--targets <locales>` | locales cibles séparées par des virgules vers lesquelles traduire (par défaut `de`) |
| `--path <pattern>` | modèle de fichier de locale contenant le jeton `{locale}` (par défaut `locales/{locale}.json`) |
| `--yes` | ignorer les invites et accepter les valeurs par défaut |
| `--force` | écraser une configuration ou un `.env.example` existant |

Le fournisseur est la seule valeur qu'`init` ne peut pas deviner. Sans terminal pour demander (en CI, ou avec `--yes`), passe `--provider`. Tout le reste revient à sa valeur par défaut lorsque tu l'omets.

## Fichiers existants

Par défaut, `init` n'écrase jamais. Si un `verbatra.config.ts` ou un `.env.example` est déjà là, il reste intact et la commande réussit tout de même. Passe `--force` quand tu veux vraiment écraser.

## Exemples

```bash
# create config + .env example, prompting for the rest
verbatra init --provider gemini

# non-interactive, accept all defaults
verbatra init --provider deepl --yes

# set the source and target locales up front
verbatra init --provider gemini --source en --targets de,es,fr

# point at a non-default locale file layout
verbatra init --provider gemini --path src/locales/{locale}.json

# scaffold into another directory
verbatra init --provider gemini --cwd packages/app

# overwrite an existing config and .env example
verbatra init --provider gemini --force
```

## Codes de sortie

| Code | Signification |
| --- | --- |
| `0` | les fichiers ont été écrits, ou ignorés sans risque car ils existent déjà (sans `--force`) |
| `2` | une erreur d'usage : aucun `--provider` n'a été fourni sans terminal pour demander, ou la configuration assemblée a échoué à la validation |

Une fois `init` terminé, définis la clé API du fournisseur et exécute [`verbatra translate`](/docs/cli/translate). Voir [le démarrage rapide](/docs) pour la procédure complète.
