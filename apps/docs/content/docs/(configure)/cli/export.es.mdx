---
title: verbatra export
description: Exporta las cadenas sin traducir a un libro de Excel con estilo para un traductor humano.
---
Escribe las cadenas que necesitan traducción en un libro de Excel con estilo que una persona pueda rellenar, y luego devolver para [`verbatra import`](/docs/cli/import). El libro tiene una hoja por locale de destino más una hoja de instrucciones. El traductor edita una sola columna; todo lo demás está bloqueado.

```bash
verbatra export
```

Por defecto, esto escribe `verbatra-translations.xlsx` en el directorio de trabajo, solo con las cadenas **faltantes y modificadas**: las mismas claves que una ejecución automatizada de `translate` enviaría a un proveedor. Consulta [Traducción humana](/docs/manual-translation) para ver el ciclo completo.

Cada fila con una traducción actual también lleva un par `Review status` / `Review reasons` de solo lectura: las propias heurísticas consultivas de verbatra para detectar una traducción que merece una segunda mirada, recalculadas de nuevo a partir de los valores actuales de origen y destino en el momento de la exportación. Nunca afectan a lo que acepta `import`. Consulta [Traducción humana](/docs/manual-translation) para ver las columnas y [Seguridad de la traducción](/docs/translation-safety) para los códigos de motivo.

## Opciones

| Opción | Descripción |
| --- | --- |
| `--cwd <path>` | resuelve la configuración y los archivos de locale desde este directorio |
| `--config <path>` | carga este archivo de configuración en lugar de buscar uno |
| `--out <path>` | escribe el libro en esta ruta (por defecto `verbatra-translations.xlsx`) |
| `--locales <list>` | subconjunto de locales de destino separados por comas (por defecto todos los configurados) |
| `--include-unchanged` | exporta también las cadenas que ya están al día (desactivado por defecto) |
| `--json` | imprime el resultado de la exportación como JSON |

## Salida

Sin `--json`, `export` imprime la ruta de salida y una línea por locale con su recuento de filas. Con `--json`, imprime el resultado de la exportación (la ruta escrita y los recuentos de filas por locale) como un único objeto JSON en stdout. Un error de toda la ejecución va a stderr, así que stdout queda limpio y canalizable con `--json`.

## Ejemplos

```bash
# write the workbook with missing and changed strings
verbatra export

# only the German and French sheets
verbatra export --locales de,fr

# include already up-to-date strings for full context
verbatra export --include-unchanged

# write to a specific path
verbatra export --out handoff/round-2.xlsx

# export a project in another directory
verbatra export --cwd packages/app

# load a specific config instead of searching for one
verbatra export --config config/verbatra.config.ts

# machine-readable result on stdout
verbatra export --json
```

## Códigos de salida

| Código | Significado |
| --- | --- |
| `0` | el libro se escribió |
| `2` | no se pudo ejecutar: un error de toda la ejecución (configuración ausente o no válida, origen ilegible, formato desconocido, un archivo de bloqueo corrupto o un locale solicitado que no está entre los locales de destino configurados), o un error de uso (incluida una lista `--locales` vacía) |

`export` nunca llama a un proveedor y no necesita ninguna clave de API. Un valor `--locales` desconocido o vacío hace que la ejecución falle con el código de salida `2` en lugar de no exportar nada en silencio, así que un error tipográfico en un nombre de locale se detecta en lugar de pasar. Consulta [`verbatra import`](/docs/cli/import) para leer de vuelta el libro rellenado.
