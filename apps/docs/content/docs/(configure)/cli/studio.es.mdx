Verbatra Studio es un panel web local sobre tu proyecto verbatra: una vista en vivo del estado de traducción, los cambios pendientes, la configuración, la desviación del archivo de bloqueo y el historial. Por defecto nunca llama a un proveedor ni escribe un archivo. Inícialo con dos flags adicionales y gana también una acción de escritura: volver a traducir una sola clave que falla (mira "Volver a traducir una clave que falla" más abajo). Inícialo con:

```bash
verbatra studio
```

<Callout title="Solo disponible como prerelease">
Verbatra Studio por ahora solo está disponible como prerelease, todavía no hay una versión estable. Para usarlo, instala `@verbatra/cli` explícitamente bajo la etiqueta `next`: `pnpm add -D @verbatra/cli@next @verbatra/studio`. `@verbatra/studio` en sí todavía no ha tenido una versión estable, así que su propia etiqueta `latest` ya trae el build de prerelease actual, sin necesidad de etiqueta. Un simple `pnpm add -D @verbatra/cli` (sin etiqueta) resuelve a la versión estable actual y no incluye este comando.
</Callout>

## Solo local, y de solo lectura por defecto

Studio se vincula únicamente a `127.0.0.1`: nunca es accesible desde otra máquina, y no hay ninguna opción para cambiar eso. Cada solicitud se comprueba contra una lista estricta de Host y Origin permitidos, y cada página y cada llamada a la API requiere el token de sesión impreso en la URL de inicio, comparado en tiempo constante para que un intento incorrecto no pueda adivinarse por temporización. Studio no envía cabeceras CORS, así que una pestaña del navegador en otro origen no podría leerlo aunque lo intentara. Por defecto no hay ningún endpoint que cambie el estado en Studio: nunca llama a `translate`, nunca ejecuta `watch` y nunca importa ni exporta un workbook. Todo lo que ves proviene de los mismos cálculos de solo lectura que [`verbatra check`](/docs/cli/check) y [`verbatra diff`](/docs/cli/diff). Pasa `--allow-spend` y `--allow-write` juntos al iniciar y Studio gana exactamente una acción de escritura, descrita en "Volver a traducir una clave que falla" más abajo; deja cualquiera de los dos flags sin activar y Studio sigue siendo igual de solo lectura.

## Referencia de comandos

| Opción | Descripción |
| --- | --- |
| `--cwd <path>` | resuelve la configuración y los archivos de locale desde este directorio |
| `--config <path>` | carga este archivo de configuración en lugar de buscar uno |
| `--port <n>` | sobrescribe el puerto por defecto de Studio (debe ser un entero de 1 a 65535) |
| `--allow-spend` | permite que Studio llame a un proveedor de traducción (también: `VERBATRA_STUDIO_ALLOW_SPEND`) |
| `--allow-write` | permite que Studio escriba archivos de locale y el archivo de bloqueo (también: `VERBATRA_STUDIO_ALLOW_WRITE`) |

`--allow-spend` y `--allow-write` están ambos desactivados por defecto. Cada uno también lee su variable de entorno equivalente cuando el flag mismo está ausente (`"1"`, `"true"`, `"yes"` o `"on"`, sin distinguir mayúsculas y minúsculas, cuenta como activado); un flag explícito de la CLI siempre gana sobre la variable de entorno.

`verbatra studio` carga `.env.local` y `.env` igual que cualquier otro comando, luego resuelve tu configuración una vez, antes de tocar siquiera la red. Una configuración ausente o no válida sale con `2` y el mismo error renderizado que cualquier otro comando; Studio en sí nunca llega a iniciarse.

## La URL impresa

Una vez que Studio está escuchando, imprime una línea en stdout:

```
Verbatra Studio running at http://127.0.0.1:5849/?token=<token>
```

Abre exactamente esa URL. El token en la cadena de consulta arranca una cookie de sesión para la pestaña del navegador; sin él (o con uno incorrecto) toda solicitud es rechazada. Nunca se imprime nada antes de que el servidor haya vinculado su puerto con éxito, así que un inicio fallido nunca filtra una URL ni un token.

## Reinicio ante un cambio de configuración

La configuración se resuelve una vez, al iniciar, y cada vista, durante toda la vida de ese proceso, se construye a partir de esa misma configuración resuelta. Si cambias `verbatra.config.ts` (un nuevo locale de destino, un proveedor distinto, un patrón de archivo distinto), detén Studio e inícialo de nuevo para aplicar el cambio. Los archivos del proyecto son distintos: Studio vigila tu archivo de locale de origen, cada archivo de locale de destino y el archivo de bloqueo, y envía una señal de actualización en vivo a tu pestaña del navegador abierta en cuanto uno de ellos cambia en disco (las escrituras sucesivas rápidas se agrupan en una sola señal). La pestaña vuelve a cargar automáticamente la vista que estás mirando en ese momento, sin necesidad de recargar la página. Si una recarga falla, la vista sigue mostrando sus últimos datos válidos con un indicador discreto de datos desactualizados en lugar de quedar en blanco, y lo intenta de nuevo en la siguiente señal. Una conexión caída se reconecta sola con espera creciente; si Studio mismo se apaga, la pestaña muestra un aviso a pantalla completa para reiniciarlo desde la terminal.

## Mensajes de error

Cuando algo sale mal, por ejemplo si no se puede leer el archivo de locale de origen, un archivo de locale de destino no se puede analizar para su formato configurado, o tu sesión ha caducado, Studio muestra una descripción concreta y útil de lo que ha pasado en lugar de un único mensaje genérico. Un error que Studio no reconoce específicamente sigue mostrándose con el mismo mensaje genérico de siempre; nunca ves una traza de pila en bruto ni un objeto de error interno.

## Las vistas

- **Estado**: los recuentos de claves faltantes, desactualizadas y al día por locale, el mismo cálculo que `verbatra check`.
- **Diff**: las claves exactas que se añadirían, se volverían a traducir o quedarían huérfanas por locale, el mismo cálculo que `verbatra diff`. Las listas de claves largas están limitadas en la vista (con una nota "mostrando N de total") para que un catálogo muy grande siga siendo responsivo; el filtrado sigue buscando en la lista completa. Un botón "Copiar como informe de revisión" convierte el diff completo, incluido todo lo que queda más allá del límite, en un resumen en Markdown en tu portapapeles, con una breve confirmación "Copiado" - listo para pegar en la descripción de un pull request o en un mensaje de chat.
- **Vista general**: una proyección redactada de tu configuración resuelta (locales de origen y destino, formato, patrón de archivo, id del proveedor y las opciones escalares seguras) más tus términos de glosario, si los hay. Los valores de texto libre que podrían contener un secreto pegado nunca se incluyen.
- **Desviación del bloqueo**: si existe un archivo de bloqueo, su versión y su desviación por locale frente a los archivos de origen y destino actuales, sin escribir en él.
- **Historial**: el historial de commits de tus archivos de locale vigilados, leído mediante `git log` con un número de commits acotado y sin `--follow`. Un proyecto sin repositorio git, o con git no disponible, muestra el historial como no disponible en lugar de un error. El historial truncado de un clon superficial tampoco es un error: simplemente muestra lo que tiene el clon.

## Paleta de comandos

Pulsa Cmd+K (Ctrl+K en Windows y Linux) en cualquier parte de Studio para abrir una paleta de comandos. Escribe para filtrar su lista: siempre ofrece las cinco vistas de arriba como destinos de salto, más, en cuanto la vista Diff haya cargado datos durante la sesión actual, cada combinación de clave y locale que se muestra ahí en ese momento. Seleccionar una vista cambia a ella. Seleccionar una clave y un locale cambia a Diff y abre directamente la vista de detalle de esa clave, la misma vista que abre un clic manual sobre esa clave. Pulsa Esc, o haz clic fuera de la paleta, para cerrarla y devolver el foco a donde estabas.

Igual que el resto de Studio, la paleta es de solo lectura: ningún elemento, en ningún estado, dispara nunca una solicitud de red, una escritura de archivo ni una llamada a un proveedor. Es pura navegación.

## Vista de detalle de una clave

En Diff, haz clic en una clave modificada para abrir su vista de detalle, con una fila por locale de destino. Cada fila también muestra un indicador de integridad de marcadores de posición e ICU: verde significa que la traducción de ese locale todavía tiene los mismos marcadores de posición que el origen (cosas como `{name}` o `{{count}}`, según tu formato); rojo significa que falta un marcador de posición o que se añadió uno inesperado, y nombra el token concreto para que sepas qué corregir. Un indicador neutro y sin etiqueta significa que el formato no tiene marcadores de posición que comprobar, o que la comprobación todavía no se ha cargado, ya que se obtiene solo al abrir la vista de detalle, no de forma anticipada. Esto solo se aplica a las claves modificadas: una clave faltante todavía no tiene un valor de destino, y una clave huérfana no tiene un valor de origen actual, así que ninguna de las dos tiene un segundo valor que comparar.

## Volver a traducir una clave que falla

Inicia Studio con `--allow-spend` y `--allow-write` juntos (o sus variables de entorno equivalentes) y la vista de detalle de la clave gana una acción de escritura: un botón "Retranslate" junto a cualquier fila de locale cuyo indicador de integridad esté en rojo. Deja cualquiera de los dos flags sin activar y el botón está ausente del panel, no simplemente desactivado.

```bash
verbatra studio --allow-spend --allow-write
```

Al hacer clic se realiza una única llamada en vivo a tu proveedor de traducción configurado para exactamente esa clave y ese locale, el mismo costo de proveedor que volver a traducir una clave con [`verbatra translate`](/docs/cli/translate). El resultado que devuelve pasa por la misma comprobación de integridad de marcadores de posición e ICU antes de escribir nada: un resultado aceptado actualiza el archivo de locale de destino y la entrada de bloqueo para esa única clave, dejando intactas todas las demás claves y locales; un resultado rechazado no escribe nada. El botón informa del resultado: "Retranslated" si tiene éxito, "Rejected: placeholder mismatch" o "Rejected: invalid message syntax" cuando el nuevo resultado en sí falla la comprobación, o "Failed: ..." para cualquier otro caso, por ejemplo si la propia llamada al proveedor falla. Una retraducción exitosa llega a tu pestaña del navegador mediante la misma señal de actualización en vivo que un cambio hecho fuera de Studio.

## Instalación

`@verbatra/studio` es el paquete propio de Studio. Todavía no ha tenido una versión estable, así que una instalación simple ya trae el build de prerelease actual, sin necesidad de etiqueta. Si no está instalado, `verbatra studio` sale con `2` y:

```
Verbatra Studio requires @verbatra/studio. Install it with: pnpm add -D @verbatra/studio
```

## Códigos de salida

| Código | Significado |
| --- | --- |
| `0` | Studio se detuvo limpiamente con un solo interrupt (Ctrl-C) |
| `2` | no se pudo ejecutar: un error de toda la ejecución (configuración ausente o no válida, `@verbatra/studio` no instalado, o el puerto ya está en uso), o un error de uso (un `--port` fuera de rango o que no es un entero) |
| `130` | detenido a la fuerza por un segundo interrupt |
