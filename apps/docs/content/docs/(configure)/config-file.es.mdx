---
verbatra lee un objeto de configuración pequeño y no secreto, y lo valida antes de hacer cualquier trabajo. Aquí no hay ninguna clave de API: el proveedor lee su clave del entorno, así que este archivo es seguro de versionar.

<Callout title="Las claves vienen del entorno">
De este archivo nunca se lee ninguna clave de API. Cada proveedor lee su clave de una variable de entorno, así que la configuración sigue siendo segura de versionar. Consulta [Proveedores](/docs/providers).
</Callout>

## Una configuración mínima

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"],
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "gemini",
    options: {
      model: "gemini-2.5-flash", // replace with your provider's model id
      maxOutputTokens: 4096,
    },
  },
});
```

`defineConfig` es un ayudante de identidad de `@verbatra/sdk`. Devuelve su argumento sin cambios y existe solo para darte inferencia de tipos y autocompletado en el editor.

<Callout title="Autocompletado de modelos en TypeScript">
Cuando escribes la configuración en TypeScript, tu editor ofrece los IDs de modelo conocidos del proveedor seleccionado para `options.model`, acotados por el `id` de proveedor que elegiste. La lista viene del propio SDK de ese proveedor (`@anthropic-ai/sdk`, `openai`, `@google/genai`), así que se mantiene al día con la versión del SDK que tengas instalada. El campo está restringido a los modelos de ese proveedor: un modelo de otro proveedor (por ejemplo, un modelo de Claude bajo `id: "gemini"`) es un error de tipo. El runtime valida `model` solo como una cadena no vacía, así que un modelo recién salido que el SDK aún no haya listado igualmente funciona en runtime aunque el editor lo marque.
</Callout>

## Campos

- **`sourceLocale`** (cadena): el locale en el que están escritas tus cadenas de origen.
- **`targetLocales`** (array de cadenas): los locales a los que traducir. No debe incluir `sourceLocale`.
- **`format`** (cadena): uno de `i18next-json`, `vue-i18n-json`, `next-intl-json`, `ngx-translate-json`, `arb`, `yaml` o `xliff`. Consulta [Formatos](/docs/formats) para saber qué detecta cada uno y cómo maneja los marcadores de posición.
- **`files.pattern`** (cadena): la ruta a cada archivo de locale. Debe contener el token `{locale}`, que verbatra sustituye por cada locale (por ejemplo, `locales/{locale}.json` se resuelve a `locales/de.json`).
- **`provider`** (objeto): el proveedor de traducción, seleccionado por `id`. Cada proveedor toma sus propias `options`, y en TypeScript el campo `model` está restringido a los IDs de modelo conocidos del proveedor elegido (un modelo de otro proveedor es un error de tipo). Consulta [Proveedores](/docs/providers).
- **`glossary`** (opcional, objeto o cadena): un mapa de términos de origen a términos de destino preferidos, o una ruta a un archivo JSON con el mismo mapa. Una ruta relativa se resuelve respecto al directorio del archivo de configuración cargado (o respecto al directorio de trabajo cuando la configuración se pasa como un override en memoria). Los proveedores LLM (Anthropic, OpenAI, Gemini y openai-compatible) reciben el mapa resuelto con la petición y se les instruye a tratar sus términos como vinculantes. DeepL no aplica este mapa de términos; registra un aviso y en su lugar aplica un glosario configurado por id (consulta [Proveedores](/docs/providers)).
- **`tone`** (opcional): uno de `"formal"`, `"informal"` o `"neutral"`. Los proveedores LLM lo reciben y se les instruye a respetarlo. DeepL lo mapea a formalidad: `formal` se vuelve más formal, `informal` se vuelve menos formal, y `neutral` o un tono ausente se omite. Con una clave DeepL de nivel gratuito, un tono distinto del predeterminado no se admite y se degrada al predeterminado con un aviso.
- **`prune`** (opcional, booleano, por defecto `false`): cuando es true, las claves presentes en un archivo de destino pero ausentes del origen (las claves huérfanas del diff) se eliminan del archivo escrito y del bloqueo. Una opción `prune` por ejecución en `translate` (el flag `--prune` de la CLI) la anula. Solo se eliminan alguna vez las claves huérfanas.
- **`generatePlurals`** (opcional, booleano, por defecto `false`): cuando es true, verbatra sintetiza las formas de plural CLDR que un idioma de destino requiere pero el origen no proporciona (por ejemplo, few/many del polaco). Se admite solo en proyectos i18next-JSON traducidos por un proveedor LLM; DeepL, los formatos no i18next y los idiomas de destino desconocidos recurren a la advertencia de plural por locale. A diferencia de `prune`, no hay flag `--generate-plurals` en la CLI, así que defínelo en la configuración; la entrada de `translate()` del SDK sí acepta una anulación `generatePlurals` por ejecución que tiene prioridad.
- **`maxBatchSize`** (opcional, entero positivo, por defecto `50`): las entradas faltantes-más-modificadas de un locale se dividen en sublotes secuenciales no mayores que este, para que una petición de proveedor sobredimensionada no pueda hacer fallar todo el locale. Un sublote fallido se retiene y se reintenta en la siguiente ejecución mientras los demás siguen avanzando. Esto es solo de configuración: no hay flag en la CLI ni anulación de `translate()` por ejecución, así que defínelo en la configuración.

Las claves de nivel superior desconocidas se rechazan. Por eso también una clave de API no puede vivir en la configuración: las claves vienen del entorno, no del archivo.

Una configuración que usa los campos opcionales:

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"],
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  glossary: {
    "Sign in": "Anmelden",
  },
  tone: "formal",
  maxBatchSize: 25,
  provider: {
    id: "anthropic",
    options: {
      model: "claude-sonnet-4-6",
      maxTokens: 4096,
    },
  },
});
```

<Callout title="Glosario como ruta de archivo">
`glossary` también acepta una ruta de cadena a un archivo JSON con la misma forma (un objeto plano de claves de cadena a valores de cadena), en lugar del objeto en línea:

```ts
export default defineConfig({
  // ...
  glossary: "glossary.json",
});
```

El archivo debe estar codificado en UTF-8 y no superar 1 MiB. Se lee una vez cuando se carga la configuración; verbatra no lo vigila en busca de cambios, así que edítalo y reinicia para aplicar una actualización. Un archivo ausente, JSON inválido o un valor que no sea un mapa plano de cadenas es un error de configuración que nombra la ruta resuelta.
</Callout>

## Orden de descubrimiento

verbatra busca hacia arriba desde el directorio de trabajo y usa la primera fuente que encuentra. Que haya varias fuentes presentes no es un error; gana la primera coincidencia. El orden de búsqueda es:

1. `package.json` (una propiedad `"verbatra"`)
2. `.verbatrarc`
3. `.verbatrarc.json`
4. `.verbatrarc.yaml`
5. `.verbatrarc.yml`
6. `.verbatrarc.js`
7. `.verbatrarc.cjs`
8. `.verbatrarc.ts`
9. `verbatra.config.js`
10. `verbatra.config.cjs`
11. `verbatra.config.ts`

Para cargar un archivo explícito en lugar de buscar, pasa `--config <path>`.

## El directorio de trabajo y .env

Ejecuta verbatra desde la raíz de tu proyecto, o pasa `--cwd` apuntando a ella. El directorio de trabajo es el único ancla para todo lo que verbatra resuelve: la búsqueda de configuración empieza ahí y sube, `.env.local` y `.env` se cargan desde ahí, y cada ruta de archivo de locale se resuelve contra él. Mantener los tres en el mismo lugar es lo que hace que un simple `verbatra translate` funcione.

La precedencia del entorno es, de mayor a menor: una variable ya definida en tu entorno real, luego `.env.local`, luego `.env`. Una variable ya presente nunca se sobrescribe.
