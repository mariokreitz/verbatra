# Eine Sprache hinzufügen

Eine Sprache hinzuzufügen ist eine einzeilige Konfigurationsänderung plus ein Übersetzungs-Lauf. verbatra sieht das neue Locale so, als würde ihm alles fehlen, daher füllt der erste Lauf es vollständig aus, und jeder Lauf danach bleibt inkrementell.

## Füge das Locale zu deiner Konfiguration hinzu

Füge es zu `targetLocales` in `verbatra.config.ts` hinzu. Es darf nur nicht dein `sourceLocale` sein:

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"], // "fr" added
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "gemini",
    options: {
      model: "gemini-2.5-flash",
      maxOutputTokens: 4096,
    },
  },
});
```

## Übersetzen

Führe es aus:

```bash
verbatra translate
```

Das neue Locale hat noch keine Zieldatei, daher behandelt verbatra es als leer und übersetzt jeden Quell-Schlüssel hinein, wobei die Datei aus `files.pattern` (hier `locales/fr.json`) geschrieben wird. Deine bestehenden Locales bleiben unberührt, es sei denn, ihre Quell-Strings haben sich geändert, weil die Lock-Datei sie immer noch als aktuell aufzeichnet.

Erst schauen, bevor du springst? Sieh dir die Schlüssel des neuen Locales zuerst in der Vorschau an:

```bash
verbatra translate --dry-run
```

## Sprachen mit mehr Pluralkategorien

Manche Sprachen benötigen mehr CLDR-Pluralkategorien, als deine Quelle hat. Englisch hat zwei (`one` und `other`); Polnisch, Arabisch und Russisch haben mehr. Standardmäßig übersetzt verbatra die in der Quelle gefundenen Pluralformen und erfindet die zusätzlichen nicht: Der Lauf ist weiterhin erfolgreich und gibt eine Pluralkategorie-Warnung pro Locale aus (Hinweis-Code `PLURAL_CATEGORIES_INCOMPLETE`), die das Locale und die fehlenden Kategorien benennt.

Damit verbatra stattdessen die fehlenden Formen synthetisiert, setze `generatePlurals: true` in deiner Konfiguration (oder übergib eine `generatePlurals`-Überschreibung pro Lauf):

```ts
export default defineConfig({
  // ...
  generatePlurals: true,
});
```

Die Generierung wird nur für i18next-JSON-Projekte unterstützt, die von einem LLM-Provider übersetzt werden. DeepL, die anderen Formate und unbekannte Zielsprachen fallen auf die Warnung zurück und schlagen niemals fehl. Generierte Formen werden wie jede Übersetzung integritätsgeprüft und in der Lauf-Zusammenfassung getrennt von den übersetzten Schlüsseln gemeldet.

## Pflege des Quell-Locales

Du pflegst immer nur das Quell-Locale von Hand. Um Strings hinzuzufügen, füge sie der Quelldatei hinzu und führe `translate` erneut aus: verbatra füllt sie in jedes Ziel, einschließlich desjenigen, das du gerade hinzugefügt hast.
