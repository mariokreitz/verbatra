# Add a language

Adding a language is a one-line config change plus a translation run. verbatra sees the new locale as missing everything, so the first run fills it completely, and every run after that stays incremental.

## Add the locale to your config

Add it to `targetLocales` in `verbatra.config.ts`. It just cannot be your `sourceLocale`:

```ts
import { defineConfig } from "@verbatra/sdk";

export default defineConfig({
  sourceLocale: "en",
  targetLocales: ["de", "fr"], // "fr" added
  format: "i18next-json",
  files: {
    pattern: "locales/{locale}.json",
  },
  provider: {
    id: "gemini",
    options: {
      model: "gemini-2.5-flash",
      maxOutputTokens: 4096,
    },
  },
});
```

## Translate

Run it:

```bash
verbatra translate
```

The new locale has no target file yet, so verbatra treats it as empty and translates every source key into it, writing the file from `files.pattern` (here `locales/fr.json`). Your existing locales stay untouched unless their source strings changed, because the lock file still has them on record as current.

Want to look before you leap? Preview the new locale's keys first:

```bash
verbatra translate --dry-run
```

## Languages with more plural categories

Some languages need more CLDR plural categories than your source has. English has two (`one` and `other`); Polish, Arabic, and Russian have more. By default verbatra translates the plural forms it finds in the source and does not invent the extras: the run still succeeds and emits a per-locale plural-category warning (notice code `PLURAL_CATEGORIES_INCOMPLETE`) naming the locale and the missing categories.

To have verbatra synthesize the missing forms instead, set `generatePlurals: true` in your config (or pass a per-run `generatePlurals` override):

```ts
export default defineConfig({
  // ...
  generatePlurals: true,
});
```

Generation is supported for i18next-JSON projects translated by an LLM provider only. DeepL, the other formats, and unknown target languages fall back to the warning and never fail. Generated forms are integrity-checked like any translation and reported separately from translated keys in the run summary.

## Maintaining the source locale

You only ever hand-maintain the source locale. To add strings, put them in the source file and run `translate` again: verbatra fills them into every target, including the one you just added.
